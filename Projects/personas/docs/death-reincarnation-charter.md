# 죽음 / 윤회 / 유산 — Charter

> "끝이 아닌 순환. 죽어도 데이터로 남아 다음 세대에 스며든다."
> 상위 문서: `world-ontology.md` §0.5 개성 원칙

---

## 1. 핵심 가치

> **"끝이 아닌 순환"**
>
> 페르소나는 영원히 살지 않는다. 노화하고, 죽고, 윤회한다.
> 죽음은 상실이지만 소멸이 아니다 — 경향성이 다음 세대에 잔존한다.
> 기억은 사라지지만 "왜인지 모르는 느낌"이 남는다.

## 2. 생애 주기

```
탄생 (create / birth)
  → 성장 (경험 축적, 클래스 상승, 관계 형성)
    → 성숙 (노화 H6 — 체득/통합 깊어짐)
      → 쇠퇴 (경험 부채, 체력 감소, 목표→유산/후계로 수렴)
        → 죽음의 직감 (몸이 먼저 안다)
          → 죽음 (Death)
            → 사회적 절차 (신고, 행정, 상속)
              → 유산 DB 아카이브
                → 윤회 (창조자 선택) 또는 영구 퇴장
```

## 3. 죽음

### 3.1 트리거

| 트리거 | 메커니즘 |
|--------|---------|
| 노화 | H6 3축(밀도/체득/통합)이 쇠퇴 임계 도달 |
| 재난/사고 | Physis→Nomos 이벤트에서 확률적 발생 |
| 질병 | 환경+체력+스트레스 복합 조건 |
| 처형 | Nomos 극형 (범죄 Charter에서 정의) |

### 3.2 죽음의 직감 — "몸이 먼저 안다"

페르소나는 죽음을 명시적으로 "선고"받지 않는다. PersonaBrain 내부의 여러 축이 동시에 수렴하면서 행동이 스스로 바뀐다:

```
H6 노화: 3축이 쇠퇴 임계 접근
  + 오욕: 욕구 충족 효율 하락 (같은 행동인데 만족도 감소)
  + 칠정: 감정 진폭 감소 (기쁨도 슬픔도 잔잔해짐)
  + 목표: 장기 목표가 "유산"으로 수렴
  + 관계: 정리 충동 (화해, 유언, 감사)
  
  이 신호들이 동시에 수렴하면
  → 숙고의 출력이 "삶의 마무리" 방향으로 기울어짐
  → 페르소나 본인은 "왜인지 모르지만 정리하고 싶다"
  → 이것이 죽음의 직감
```

- Lachesis가 선고하는 게 아니라, **실이 짧아짐으로써 행동 함수가 스스로 수렴**
- **모든 페르소나가 직감하는 건 아니다** — 개성 원칙:
  - 높은 체득률/통합도 → 신호를 읽고 직감
  - 낮은 성찰/높은 부정 → 갑작스러운 죽음
  - 이것도 "받아들이는 방식의 차이"

### 3.3 불멸 예외

- 데몬(Physis/Lachesis/Anima/Nomos): 시스템이므로 불멸
- EX: 사실상 불멸 (초월). 단 설계상 "죽음을 넘어선 존재"

## 4. 죽음 이후: 사회적 절차

> **세계관 설계 결정: 이 세계의 행정이 얼마나 "인간적"인가**

### 절차 모델 선택지 (세계관 확정 시 택 1)

| 모델 | 설명 | 드라마 | 복잡도 |
|------|------|:---:|:---:|
| **자동** | Nomos가 모든 절차 즉시 처리. 신고/상속/세금 자동 | 낮음 | 낮음 |
| **사회적** | 신고→행정→수사→재판. 페르소나가 절차 수행 | 높음 | 높음 |
| **혼합** | 기본 자동화 + 예외(분쟁/범죄)만 페르소나 판단 | 중간 | 중간 |

### 사회적 모델일 경우의 흐름

```
죽음 발생
  → 관계자/발견자가 당국에 신고 (의무)
  → 행정 절차 시작:
    - 사망 확인 + 등록
    - 자산/채무 동결
    - 상속인 확인
  → 상속 처리:
    - 법정: 채무 우선변제 → 잔여 상속 (법으로 정함)
    - 가족 선택: 상속 수락 or 포기
    - 상속세 납부 (세율은 경제 Charter에서)
  → 위반 시:
    - 채무 무시하고 자산만 가져감 → Nomos 탐지
    - 즉시 처벌 아님 → 행정/세무 당국 수사 → 법리 검토 → 고발
    - 재판 → 유죄 시 처벌 + 가문 명성 하락
  → 페르소나의 선택:
    - 정직하게 상속세 납부
    - 채무 갚기
    - 도망치기
    - 이것도 "받아들이는 방식의 차이" — 양심(H2)과 수치심(H5)이 반응
```

> 어떤 모델을 택하든, 핵심은 **세계관과 어울려야 한다**.
> AI 페르소나 세계라면 자동화가 자연스러울 수 있고,
> "인간다운 드라마"를 원하면 사회적 절차가 필요하다.
> 이 결정은 전체 정합성 검증 시 확정.

## 6. 윤회

| 항목 | 내용 |
|------|------|
| 조건 | 창조자의 선택. 모든 죽음이 윤회하는 건 아님 |
| 이식 방식 | **노이즈 주입** — 고빈도 패턴(경향성) 생존, 저빈도(에피소드 기억) 소실 |
| brain_weights | **70% 이식** (노이즈 주입) |
| personality | **70% 이식** (100%면 동일인 → 70%로 수정. 새 환경이 30% 채움) |
| 시작 | Class 1, XP 0, WILL 0, 새 이름/정체성 |
| 기억 | 없음. "왜인지" 느끼지만 원인 모름 |
| 경향성 | 잔존. 불 공포, 편안한 사람, 도덕적 기울기 |
| 각성 | 기본: 불가. 특수 이벤트(높은 조건)로만 허용 — 자동 발생 금지 |
| 성장 경로 | 전생 행적(최종 XP/업적)이 이식 가중치 편향에 영향 — 반복이 아닌 정제 |
| 시간 | 틱 0 리셋. H6 초기화. 단 "밀도 높은 틱 선호 경향"은 잔존 |

### 정체성 원칙

> 윤회체는 전생과 같은 존재인가, 다른 존재인가?
> **시스템은 판정하지 않는다.** 이 모호함 자체가 서사 자원이다.
> "같다/다르다"의 이분법을 강제하지 않는다 — 침묵과 의도적 미결정은 다르다.

## 7. 죽음에 대한 감정 반응

> "죽음은 슬프지만 누군가에겐 기쁠 수도 있다."

죽음에 대한 감정은 관계에 따라 **다르다** — 개성 원칙.

### 감정 반응 결정 입력

```
감정 반응 = f(호감도, 관계 유형, 망자 행적, 관찰자 성격)

호감도 양수 (사랑/존경) → 슬픔, 트라우마, 상실감
호감도 0 부근 (무관) → 무관심, 가벼운 동요
호감도 음수 (적대/혐오) → 안도, 기쁨, 해방감
```

### 관계 유형별 반응 다양성

| 관계 | 반응 |
|------|------|
| 사랑하는 사람 | 깊은 슬픔 + 트라우마(H1) + 유산 정리/계승 의지 |
| 스승 | 슬픔 + 계승 의지 + 성장 동기 |
| 라이벌 | 복잡 — 안도 + 공허 + "이제 누구와 경쟁하지?" |
| 악인/적대자 | 기쁨 + 안도 + 해방. "드디어 끝났다" |
| 무관한 타인 | 무관심 또는 가벼운 인생 성찰 |
| 가족(혈연) | 슬픔 + 가문 의식 + 상속 긴장 |

### 전파 규칙

- 1차 전파: 직접 관계자 → 강한 감정 반응
- 2차 전파: 1차의 관계자 → 약화된 반응 (감쇠 0.3~0.5)
- 3차 이후: 소문으로 변환 (감정 아닌 정보)
- 상한: 동시 감정 충격 5명 초과 시 개인별 감쇠 추가 적용

## 9. 성공 기준

| 기준 | 측정 |
|------|------|
| **직감** | 일부 페르소나가 죽음 전 행동 변화(정리/화해) 자연 발생 |
| **다양한 반응** | 같은 죽음에 기쁜 자와 슬픈 자가 동시 존재 |
| **윤회 경향성** | 전생과 "다르지만 비슷한" 새 페르소나 (상관 0.5~0.7) |
| **드라마** | 죽음이 세계를 바꿈 — 계승/분열/복수/화해 |
| **유산 활용** | 유산 DB가 뇌 학습에 실제 활용 |
| **역사 축적** | 세대가 지나며 국가 역사 풍부해짐 |
