## 1. 핵심 가치

> **"왜 오늘 비가 오는지 대답할 수 있는 세계"**
>
> 날씨는 주사위가 아니라 물리에서 나온다. 지형이 바람을 만들고, 바람이 구름을 옮기고, 구름이 비를 내린다.
> 이 인과 체인이 감정을 바꾸고, 경제를 흔들고, 재난을 일으킨다.
> 페르소나는 날씨를 수치가 아닌 **경험**으로 느낀다 — "견딜 수 없는 무더위", "우울한 장마".

## 2. 핵심 루프

```
매 틱 (게임 1시간 = 현실 30분):

  [Stage 1 — Physis ∥ Lachesis 병렬]

  궤도(LUT 조회) → 일사량
    → 기압차(지형+해류+토지피복 보정)
      → 바람(코리올리+지형 차폐)
        → 수분 이류(해양 온도+해류 보정)
          → 기온(고도 감률 6.5°C/km)
            → 날씨 확정 (기온/강수/풍속/풍향/습도/운량/체감온도)
              → 누적 버퍼 갱신 (가뭄/한파/장마 시계열)
                → 이벤트 발행 (임계 초과 시 재난 트리거)
                  → Anima 입력 (감정/활동 영향)
                  → Nomos 입력 (경제/재난 영향)
```

## 3. 5W + Who

| 항목 | 내용 |
|------|------|
| **What** | 물리 기반 기후 시뮬레이션. 궤도→일사량→바람→수분→기온→날씨 인과 체인 |
| **Why** | "주사위가 아닌 인과". 날씨가 감정·경제·재난에 연쇄 영향 |
| **Where** | 3권역: Claude(내륙 고원), Codex(해안 평야), Gemini(열대 군도) |
| **When** | 매 틱(게임 1시간). 계절 90일 주기. 일사량 LUT 사전 계산 |
| **Who** | Daemon Registry가 관리. 창조자가 최종 권한(재시작/교체/칙령 개입) |
| **Constraint** | Lv.1~2 경량, CPU only, ≤10ms/틱, JSON 기반, 3권역 셀 모델 |

**기후 모델: 계층적 셀 (Hierarchical Cell)** — /discuss 3회 + /submix 검증 확정

```
Level 0: 3권역 셀 (Physis 메인 계산)
  → 권역별 기온/강수/풍속 기본값. O(3), ~2ms
  → 실제 기상학의 종관 규모(synoptic scale) 대응

Level 1: 영지 셀 (후처리 Δ 보정)
  → 영지_날씨 = 권역_날씨 + Δ(고도, 수계 거리, 지형 차폐)
  → 영지마다 고유 보정 → 같은 Claude 안에서도 날씨가 다름
  → 경계 보간: 인접 영지 가중 평균 (온도 절벽 방지)
  → 실제 기상학의 중규모(mesoscale) + 국지 보정 대응
  → 영지당 O(1), 40영지 = ~0.1ms
```

**단일 좌표계**: 450,000km² 전체가 하나의 위경도 좌표계. 거주 면적도 같은 좌표. 스케일 변환 없음.
