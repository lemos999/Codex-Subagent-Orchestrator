# 틱 데몬 파이프라인 — Charter

> Phase B 인프라. "사람이 보지 않아도 세계는 돌아간다."
> 상위 문서: `world-ontology.md` §3 Daemon Registry

---

## 1. 핵심 가치

> **"사람이 보지 않아도 세계는 돌아간다"**
>
> 매 30분(현실) = 1시간(게임)마다 4개 데몬이 파이프라인으로 세계를 구동한다.
> 창조자가 자리를 비워도 페르소나는 살고, 날씨는 변하고, 경제는 흐른다.
> LLM 호출 0건. CPU만으로. 24시간 무중단.

## 2. 상위 구속

- World Ontology §0.3 **상위 구속**: 파이프라인 순서는 물리 법칙. 위반 불가.
- World Ontology §0.3.1 **데몬 응징**: 규칙 위반 페르소나에 대한 4단계 대응.
- World Ontology §0.2 **파이프라인 실행**: 역방향 호출 금지. 동일 틱 내.
- Physis Charter: Stage 1 기후 출력 → 이후 Stage 입력 조건.

## 3. 핵심 루프

```
매 틱 (30분 현실 = 1시간 게임):

  Stage 1: [Lachesis 선행 → Physis 후행] (반동기)
    Lachesis → 틱 번호 발행 + 시간 진행 + 크론잡 트리거 (L1b 시계 쓰기)
    Physis  → 발행된 틱 기준으로 기후 계산 (L1b 날씨 쓰기)
    (Lachesis가 틱 번호를 먼저 확정해야 Physis가 "지금"을 알 수 있음)
    (inject_anomaly 플래그가 있으면 이상 기후 주입 + 인접 영지 감쇠 전파)

  Stage 2: Anima
    → PersonaBrain 추론 (전 페르소나)
    → 감정/관계 갱신 (L3 쓰기)
    → Action_Proposal 생성 (L4 쓰기)
    → 시간 초과 시 근사 강등 (멈추지 않음)

  Stage 3: Nomos
    → Action_Proposal 검증 → 승인/거부 (L4 Action_State 쓰기)
    → WILL/gold 트랜잭션 (L6 쓰기)
    → 법 위반 탐지 + 데몬 응징 등급 판정
    → 이벤트 생성·발행 (L8 쓰기)

  Post: Event Bus
    → 이벤트 → 대시보드 WebSocket / 외부 웹훅 / 로그
    → Heartbeat 상호 확인
```

## 4. 데몬 응징 시스템

파이프라인 내에서 데몬 응징이 작동하는 흐름:

```
Stage 2 (Anima):
  PersonaBrain 출력 → **Arbiter(규칙 기반 필터)**가 1차 판별
    → 금지 행동 패턴 매칭 → violation_flag + 등급 힌트 태깅
    → Anima는 감정 처리만 (판단과 감정 분리)

Stage 3 (Nomos):
  violation_flag 감지 → 등급 **확정** 판정
    경미(캐릭터 일관성 이탈): Reject + 경고 이벤트. 끝.
    중대(세계관 파괴): Reject + Lachesis에 페널티 요청 (다음 틱 행동 제한)
    금기(시스템 무결성 훼손): Reject + 4데몬 연쇄 예약 (다음 틱 발동)
      → 금기 누적 카운터 +1 (30일 쿨다운 내 재시도 시 누적, 3회→영구 제한)

다음 틱 Stage 1:
  금기 예약이 있으면:
    Lachesis: 틱 번호 선행 발행 (병렬 동기화 기준)
    Physis: **inject_anomaly 입력** 수신 → 해당 영지 이상 기후 (ash_density 주입 등)
      → 인접 영지에 감쇠 전파 (거리 비례 감쇠율, 물리적 자연스러움)
    Lachesis: 해당 페르소나 시간 왜곡 플래그 (틱 간격 2배 = 행동 빈도 절반)

다음 틱 Stage 2:
  Anima: 왜곡 플래그 → 공포/불안 감정 주입 + 트라우마 생성
  → 같은 영지 다른 페르소나도 이상 기후 환경 패널티 수신

다음 틱 Stage 3:
  Nomos: 경제 페널티 + 이벤트 로그 확정
```

> 응징은 **같은 틱이 아닌 다음 틱에 발동** — 역방향 호출 금지 원칙 준수.
> 페르소나는 "세계가 반응한다"고 느끼지, 데몬의 존재를 인식하지 못한다.

## 5. 스코프

| 포함 | 제외 |
|------|------|
| 4-Stage 파이프라인 엔진 | 각 Executor 내부 로직 (Physis 수식, Brain 모델) |
| Heartbeat 상호 감시 (4×4) | PersonaBrain 구현 |
| 상태 내구성 (WAL 30초, 원자적 쓰기, 복구) | 대시보드 UI 구현 |
| 이벤트 버스 (Pub/Sub, WebSocket) | 외부 웹훅 구체 엔드포인트 |
| 멀티세션 격리 (LLM 독립) | LLM 통합 구현 |
| 크론잡 스케줄러 (세금/급여/생활비) | 각 크론잡의 경제 공식 |
| 데몬 응징 파이프라인 (등급 판정, 연쇄 예약) | 응징 세부 수치 (페널티 틱 수 등) |
| 프로세스 관리 (기동/정지/pm2) | OS 레벨 인프라 |

## 6. 성공 기준

| 기준 | 목표 |
|------|------|
| **자율 운행** | 창조자 부재 시 24시간+ 무중단 |
| **파이프라인 순서** | 물리적으로 위반 불가능 (Stage N-1 미완료 → Stage N 불시작) |
| **장애 복구** | 1 Executor 다운 → 60초 내 자동 재시작 |
| **상태 복구** | 크래시 후 최대 1틱 손실 (WAL 30초) |
| **LLM 비의존** | 틱 루프 내 API 호출 0건. CPU 추론만 |
| **이벤트 지연** | 틱 완료 → 이벤트 발행 ≤ 1초 |
| **응징 정합** | 역방향 호출 없이 다음 틱 발동. 인과 추적 가능 |

| D4 | 이벤트 버스 | Pub/Sub. 이벤트 유형별 채널. WebSocket + jsonl 로그. | D1 |
| D5 | 크론잡 스케줄러 | Lachesis 하위. 게임일/주/월/분기/연 기반 트리거. | D1, Lachesis |

| D8 | 응징 엔진 | Arbiter(규칙 필터) → Nomos(등급 확정) → 연쇄 예약 → 다음 틱 발동. 누적 카운터(3회→영구) | D1, D4 |
| **D9** | **Arbiter (규칙 필터)** | PersonaBrain 출력의 violation 1차 판별. 금지 패턴 매칭. Anima와 분리 (판단≠감정) | D1 |
| **D10** | **감사 로그 (Audit Trail)** | 모든 응징 판정·크론잡·에포크 전환의 불변 기록. 금기 쿨다운 30일 추적 | D3, D4 |

## 8. 리스크

| # | 리스크 | 영향 | 대응 |
|---|--------|------|------|
| R1 | 파이프라인 Stage 지연 누적 | 틱 간격 초과 → 시뮬 지연 | 근사 강등 (Anima) + 10ms 예산 모니터링 |
| R2 | WAL 파일 손상 | 상태 복구 불가 | 이중 WAL + 일일 풀 스냅샷 |

| R4 | 이벤트 버스 폭주 | 대시보드 마비 | 이벤트 throttle (초당 100건 상한) |
| R5 | 응징 연쇄가 과도 | 페르소나 플레이 불능 | 금기 30일 쿨다운 + 누적 3회→영구. 이상 기후는 인접 영지 감쇠 전파 |
| **R6** | **LLM 세션과 JSON 동시 접근** | **레이스 컨디션** | **D3 파일 락(세마포어). LLM(/persona)은 락 경유 필수** |
| **R7** | **에포크 경계 처리 누락** | **채굴 레이트 전환 실패** | **에포크 전환 시 전체 스냅샷 + 이벤트 + Nomos 레이트 갱신** |

## 9. 특수 이벤트

### 에포크 전환
- Lachesis가 누적 채굴량 감시 → 에포크 경계 도달 감지
- 에포크 전환 절차:
  1. 현재 틱 완료 후 **전체 상태 스냅샷** (D3)
  2. **에포크 전환 이벤트** 발행 (D4) → 전 페르소나 알림
  3. Nomos가 **채굴 레이트 갱신** (반감)
  4. 감사 로그 기록 (D10)
