# 코인 전략 명세서

## 문서 목적
이 문서는 ETHUSDT 자동매매 전략을 개발자가 그대로 구현할 수 있도록 규칙, 상태 전이, 진입/청산/무효화 조건, 예외 처리, 개발 체크리스트를 한 번에 정리한 명세서다.

---

## 1. 전략 철학

### 1.1 핵심 원칙
- 예측보다 **조건부 대응**을 우선한다.
- 진입 이유보다 **무효화와 손절 구조**를 먼저 정의한다.
- 상위 타임프레임이 하위 타임프레임보다 우선한다.
- 박스 중앙 추격은 금지한다.
- **상위 추세 강세 + 하위 조정 종료 확인**에서만 공격적으로 진입한다.
- **하위 타임프레임 약세 상태에서의 롱 추격 금지**를 기본 규칙으로 둔다.

### 1.2 전략 목적
- ETHUSDT를 기준으로 **상승 추세 내 눌림 매수 / 반등 실패 숏** 두 가지를 모두 자동화한다.
- 단, 시스템 기본 성향은 **상위봉 강세 편향**이다.
- 즉, 숏은 메인 전략이 아니라 **조정 연장 대응 전략**이다.

## 2. 사용 지표 및 타임프레임

### 2.1 필수 타임프레임
- **4시간봉**: 상위 구조 판정
- **1시간봉**: 메인 방향과 핵심 지지/저항 판정
- **30분봉**: 실행 구간 판정
- **15분봉**: 진입 직전 확인
- **5분봉**: 트리거 발생 확인

### 2.2 필수 지표
- 이치목 기본 구성
  - Price vs Cloud
  - Tenkan
  - Kijun
  - TK State (Tenkan > Kijun / Tenkan < Kijun)
- Volume Profile Visible Range
  - POC
  - VAH
  - VAL
- 거래량
- 선택: Fib 확장 레벨

### 2.3 핵심 데이터 필드
각 타임프레임에서 아래 값을 매 바마다 계산한다.

- `close`
- `tenkan`
- `kijun`
- `cloud_position` = above / in / below
- `tk_state` = bullish / bearish
- `poc`
- `vah`
- `val`
- `profile_bias` = above_va / inside_va / below_va
- `volume`
- `volume_ma_short`
- `volume_ma_long`

## 3. 상위 구조 판정 로직

### 3.1 4시간봉 강세 판정
다음 조건을 모두 만족하면 `HTF_BULLISH = true`

1. `4h.close > 4h.cloud_top`
2. `4h.tenkan > 4h.kijun`
3. `4h.close >= 4h.vah` 또는 최소 `4h.profile_bias != below_va`

### 3.2 4시간봉 중립 판정
다음 중 하나면 `HTF_NEUTRAL = true`
- 4시간 가격이 구름 안
- Tenkan > Kijun 이지만 가격이 구름 상단 바로 위/아래에서 흔들림
- 4시간이 above_va지만 최근 2~3봉 연속 상단 거절

### 3.3 4시간봉 약세 판정
다음 조건을 모두 만족하면 `HTF_BEARISH = true`

1. `4h.close < 4h.cloud_bottom`
2. `4h.tenkan < 4h.kijun`
3. `4h.profile_bias == below_va`

## 4. 기본 운영 모드

### 4.1 모드 분류
시스템은 매 시점 아래 4개 중 하나의 모드만 가진다.

- `MODE_TREND_LONG`
- `MODE_PULLBACK_LONG`
- `MODE_REBOUND_SHORT`
- `MODE_NO_TRADE`

### 4.2 모드 전환 규칙

#### MODE_TREND_LONG
조건:
- 4시간 강세
- 1시간 강세 또는 최소 1시간 중립
- 30분이 구름 위 + TK 강세
- 15분/5분 눌림 후 재상승

#### MODE_PULLBACK_LONG
조건:
- 4시간 강세
- 1시간 중립 또는 구름 안
- 30분/15분이 조정 상태
- 핵심 지지대 접근 중

#### MODE_REBOUND_SHORT
조건:
- 4시간 강세가 약화 중이거나 중립
- 1시간/30분/15분 모두 약세 정렬
- 반등이 주요 저항(30분 Kijun, 1시간 Tenkan/Kijun, VAH)에서 실패

#### MODE_NO_TRADE
조건:
- 상위/하위 방향 충돌이 심함
- 박스 중앙
- 손절 거리 과도
- 급등 직후 이미 확장 과열 구간

## 5. 핵심 가격대 정의

자동매매는 고정 숫자를 하드코딩하지 않고 아래 우선순위로 레벨을 계산한다.

### 5.1 상단 저항 우선순위
1. 30분 VAH
2. 1시간 VAH
3. 30분 Kijun
4. 1시간 Kijun
5. 최근 스윙 고점
6. Fib 확장 2.5 / 3.618 / 4.0

### 5.2 하단 지지 우선순위
1. 30분 Tenkan
2. 1시간 Tenkan
3. 30분 Kijun
4. 1시간 Kijun
5. 30분 POC
6. 30분 VAL
7. 1시간 POC
8. 1시간 VAL
9. 4시간 Kijun

### 5.3 지지/저항 존 생성 규칙
레벨은 단일 숫자가 아니라 구간으로 처리한다.

- `zone_width = max(current_price * 0.0015, ATR_15m * 0.25)`
- 특정 레벨 `L`에 대해 존은 `[L - zone_width, L + zone_width]`

예시:
- `2322~2320`처럼 인접 레벨이 붙어 있으면 하나의 합성 구간으로 병합

## 6. 롱 전략 상세

### 6.1 추세 지속 롱 (Trend Continuation Long)

#### 진입 전제
- 4시간 강세
- 1시간 강세 또는 중립
- 30분이 구름 위
- 15분 또는 5분에서 눌림 후 반등

#### 진입 조건
아래 조건을 모두 만족할 때 진입

1. 가격이 30분 Tenkan 또는 1시간 Tenkan 존에 접근
2. 5분봉에서 아래꼬리 발생 또는 직전 저점 갱신 실패
3. 15분봉 종가가 다시 5분 반등 레벨 위
4. 30분봉이 최소 지지 존 위에서 마감 예정 또는 이미 마감 완료
5. 반등 봉 거래량이 직전 5개 평균 이상

#### 초기 손절
아래 중 더 아래값 사용
- 최근 5분 스윙 저점 - `0.2 * ATR_5m`
- 진입에 사용된 지지 존 하단 - `0.2 * ATR_15m`

#### 1차 익절
- 최근 고점 또는 30분 VAH

#### 2차 익절
- 1시간 VAH 또는 Fib 다음 확장값

#### 트레일링
1차 익절 후 잔량은
- 5분 Kijun 하향 이탈 시 절반 축소
- 15분 Tenkan 하향 이탈 시 전량 청산

---

### 6.2 눌림 매수 롱 (Pullback Long)

#### 진입 전제
- 4시간 강세 유지
- 1시간 강세 약화 또는 구름 안
- 30분/15분 약세 조정 중
- 핵심 지지대 접근: 30분 Kijun / 1시간 Kijun / POC / VAL / 4시간 Kijun

#### 진입 조건
1. 가격이 핵심 지지 존 도달
2. 15분봉에서 하단 이탈 실패 또는 망치형/양봉 전환
3. 5분봉 고점이 이전 3개 봉 최고가 돌파
4. 15분 거래량이 전 봉 대비 증가
5. 30분봉 종가 기준 해당 지지 존을 회복 또는 유지

#### 손절
- 15분 기준 지지 존 하단 이탈 + 15분 종가 확정
- 또는 1시간 종가가 핵심 지지 하단 아래 확정

#### 청산
- 1차: 30분 Tenkan
- 2차: 30분 Kijun
- 3차: 1시간 Tenkan / Kijun / POC

---

## 7. 숏 전략 상세

### 7.1 반등 실패 숏 (Rebound Failure Short)

#### 진입 전제
- 30분 below_cloud 또는 inside_va + TK 약세
- 15분 below_cloud + TK 약세
- 5분 below_va or below_cloud
- 4시간 강세더라도 하위봉이 충분히 약하면 단기 숏 허용

#### 진입 조건
1. 급락 후 기술적 반등 발생
2. 반등이 아래 저항 중 하나에서 멈춤
   - 15분 Kijun
   - 30분 Tenkan
   - 30분 Kijun
   - 30분 VAH / POC
   - 1시간 Tenkan
3. 5분에서 윗꼬리 + 음봉 전환
4. 15분이 저항 구간 아래에서 마감
5. 거래량이 반등보다 재하락 시 더 강함

#### 손절
- 반등 저항 존 상단 + `0.2 * ATR_15m`

#### 목표
- 1차: 직전 저점
- 2차: 30분 POC
- 3차: 30분 VAL
- 4차: 1시간 VAL 또는 4시간 Kijun

#### 청산 규칙
- 5분 Kijun 상향 돌파 시 일부 청산
- 15분 Tenkan 회복 시 절반 축소
- 15분 Kijun 회복 시 전량 청산

---

## 8. 무효화 규칙

### 8.1 롱 무효화
아래 중 하나면 롱 아이디어 무효
- 사용한 지지 존 하단 아래 15분 종가 확정
- 더 중요한 상위 지지 아래 1시간 종가 확정
- 30분/15분/5분이 모두 below_cloud + TK 약세로 정렬

### 8.2 숏 무효화
아래 중 하나면 숏 아이디어 무효
- 사용한 저항 존 상단 위 15분 종가 확정
- 30분이 다시 구름 위 + TK 강세로 복귀
- 1시간이 POC/VAH 위로 회복하고 유지

## 9. 포지션 관리

### 9.1 기본 리스크
- 1회 거래 최대 손실: 계좌의 `0.5% ~ 1.0%`
- 상위 강세 역추세 숏은 최대 손실 `0.3% ~ 0.5%`

### 9.2 분할 진입
- 1차 진입 50%
- 확인 후 추가 30%
- 돌파/회복 확인 후 추가 20%

### 9.3 분할 청산
- 1차 목표 도달 시 30%
- 2차 목표 도달 시 30%
- 잔량 40%는 트레일링

### 9.4 동시 포지션 제한
- 동일 방향 중복 포지션 최대 2개
- 반대 방향 포지션 동시 보유 금지

## 10. 금지 조건

다음 상황에서는 신규 진입 금지.

1. 박스 중앙에서 방향 없는 횡보
2. 직전 3봉 평균 대비 거래량 급감
3. 손절 거리 대비 목표가 1차 보상비가 1.5 미만
4. 경제 이벤트 직전/직후 비정상 급변동 구간
5. 상위봉과 하위봉이 완전히 충돌하는데 명확한 지지/저항이 없는 구간
6. 급등 직후 최고가 근처에서 신규 추격 롱
7. 급락 직후 최저가 근처에서 신규 추격 숏

## 11. 상태별 실행 규칙

### 11.1 4시간 강세 + 1시간 강세 + 30분 강세
- 모드: `MODE_TREND_LONG`
- 허용: 눌림 매수, 돌파 확인 매수
- 금지: 숏

### 11.2 4시간 강세 + 1시간 중립 + 30분 약세
- 모드: `MODE_PULLBACK_LONG`
- 허용: 핵심 지지 존에서만 롱
- 허용: 저항 실패 시 짧은 숏
- 금지: 시장가 추격 롱

### 11.3 4시간 중립 + 1시간 약세 + 30분 약세
- 모드: `MODE_REBOUND_SHORT`
- 허용: 반등 실패 숏
- 허용: 매우 강한 지지 확인형 롱만 제한적

### 11.4 4시간 약세
- 모드: `MODE_REBOUND_SHORT`
- 허용: 반등 실패 숏 중심
- 금지: 롱 추격

## 12. 구현용 의사코드

```text
if HTF_BULLISH:
    if 30m_above_cloud and 30m_tk_bullish:
        if pullback_to(30m_tenkan or 1h_tenkan) and lower_tf_reversal_confirmed:
            enter_long()
    elif 30m_below_cloud or 15m_below_cloud:
        wait_for_support_zone()
        if support_holds and reversal_confirmed:
            enter_long_small()
        elif rebound_to_resistance_fails:
            enter_short_small()

if HTF_NEUTRAL:
    if 30m_15m_5m_bearish_alignment and rebound_fails:
        enter_short()
    elif major_support_holds and 30m_recovers:
        enter_long()

if HTF_BEARISH:
    if rebound_to_resistance_fails:
        enter_short()
```

## 17. 최종 핵심 문장

이 전략은 **상위 구조는 4시간, 방향은 1시간, 실행은 30분, 진입 트리거는 15분과 5분**으로 나눈다. 상위봉이 살아 있으면 하위 조정에서만 롱을 노리고, 하위봉이 전부 약세 정렬이면 반등 실패 숏을 제한적으로 운용한다. 모든 진입은 손절 기준이 먼저 확정되어야 하며, 지지/저항은 단일 숫자가 아니라 구간으로 처리한다.

## 18. 검증 및 진화 운영 원칙

### 18.1 최종 목표
이 전략 문서의 최종 목표는 단순 백테스트 수익률 극대화가 아니다.

최종 목표는 아래와 같다.

- 가상 머니와 실시간 검증을 통해 **시장 레짐별로 생존 가능한 자동매매 시스템**을 만든다.
- 검증된 규칙만 실전에 투입해 **장기 기대값이 플러스인 구조**를 구축한다.
- 특정 구간 최적화가 아니라 **붕괴하지 않고 적응 가능한 시스템**을 만든다.

### 18.2 검증 철학
- 수익보다 먼저 **생존**을 본다.
- 진입보다 먼저 **무효화**를 본다.
- 백테스트 수익률보다 먼저 **데이터 정합성**을 본다.
- 높은 승률보다 **기대값과 최대 낙폭**을 우선한다.
- 하나의 만능 전략보다 **레짐별 성능이 분리된 적응형 구조**를 선호한다.

### 18.3 전략 구성 분리
전략은 아래 4개 층위로 분리한다.

1. **아이디어 레이어**
   - 어떤 상황에서 진입할 가치가 있는가
2. **실행 레이어**
   - 언제, 어떤 가격 기준으로, 얼마를 진입하는가
3. **리스크 레이어**
   - 틀렸을 때 언제, 얼마나 작게 잘라낼 것인가
4. **검증 레이어**
   - 이 전략이 우연이 아니라 반복 가능한가

위 4개가 섞이면 시스템이 아니라 감정 자동화가 된다.

## 19. 검증 파이프라인

### 19.1 1단계: 규칙 기반 시스템화
모든 규칙은 사람이 해석하는 문장이 아니라 코드로 번역 가능한 조건문이어야 한다.

예시:
- 4시간봉이 구름 위이고 Tenkan > Kijun일 때만 롱 허용
- 1시간봉이 Kijun 위일 때만 추세 지속으로 판정
- 15분봉 눌림 후 5분봉 반전 캔들에서 진입
- 손절은 15분 최근 스윙로우 하단 또는 Kijun 이탈
- 1차 부분익절 후 조건부 트레일링 적용

### 19.2 2단계: 이벤트 기반 백테스트
단순 봉 종가 기준 시뮬레이션만으로는 실전 재현성이 떨어진다.
백테스트 엔진은 아래를 반영해야 한다.

- 슬리피지
- 수수료
- 부분 체결
- 주문 실패 및 재시도
- 급변동 구간에서의 비정상 체결
- 동일 시점 신호 중복 처리

### 19.3 3단계: 워크포워드 검증
파라미터 최적화는 전체 구간에 맞추지 않는다.

필수 분할:
- 학습 구간
- 검증 구간
- 완전 미사용 테스트 구간

원칙:
- 학습 구간에서만 파라미터 튜닝
- 검증/테스트 구간에서는 고정 파라미터 유지
- 새 구간 성능이 크게 무너지면 과최적화로 간주

### 19.4 4단계: 실시간 페이퍼 트레이드
백테스트 통과 후 바로 실전에 가지 않는다.
먼저 실시간 페이퍼 트레이드를 운영한다.

검증 항목:
- 실시간 봉 마감 처리 정확성
- API 지연과 재연결 안정성
- 주문 시뮬레이션과 실제 시장 체결 차이
- 실시간 상태 전이 로그 정확성
- 전략 신호 빈도 및 중복 진입 문제

### 19.5 5단계: 초소액 실전
실전 전환 초기에는 수익보다 실행 오차 검증이 우선이다.

원칙:
- 최소 주문 단위 또는 총자산의 극소 비율만 사용
- 실전 슬리피지와 수수료 재계산
- 페이퍼 대비 체결/청산 차이 로그 축적
- 최소 2~4주 이상 운영 후 증액 여부 판단

## 21. 적응형 시스템 설계 원칙

### 21.1 고정 전략이 아니라 레짐 적응형
단일 전략 하나로 모든 장을 커버하려 하지 않는다.
아래 3개 엔진을 분리 설계한다.

1. **상위 방향 필터 엔진**
   - 4시간/1시간 기반
   - 롱만 허용 / 숏만 허용 / 노트레이드 판정
2. **진입 엔진**
   - 눌림 매수 / 돌파 추종 / 반등 실패 숏
3. **청산 엔진**
   - 부분익절 / 기준선 이탈 축소 / 트레일링 / 전체 청산

### 21.2 좋은 전략보다 좋은 청산이 중요
자동매매는 진입보다 청산에서 성과 차이가 크게 난다.
따라서 청산 엔진은 별도 실험 대상으로 관리한다.

비교 항목:
- 고정 R 익절
- 저항/지지 기반 익절
- Kijun / Tenkan 기반 트레일링
- ATR 기반 트레일링
- 부분익절 후 잔량 추세 추종

### 21.3 과최적화 방지
- 파라미터 수를 최소화한다
- 조건을 지나치게 세분화하지 않는다
- 특정 구간 수익률만 보고 규칙을 추가하지 않는다
- 신호 수가 지나치게 줄어드는 필터는 경계한다

## 22. 방어 장치 및 운영 중단 규칙

### 22.1 거래 중단 조건
아래 중 하나면 시스템은 신규 진입을 중단한다.

- 연속 4손실 이상
- 일일 손실이 -3R 이하
- 주간 손실이 허용 한도 초과
- API 오류/주문 실패가 연속 N회 발생
- 슬리피지가 기준치 이상 급증
- 비정상 변동성 급등으로 스프레드가 확대됨

### 22.2 포지션 크기 제한
- 1회 거래 리스크: 총자산의 `0.25% ~ 0.5%` 권장
- 동시 활성 포지션 총 위험: 총자산의 `1.0%` 이내
- 역추세 숏은 기본 리스크의 50% 이하
- 고변동 구간에서는 포지션 크기 자동 축소

### 22.3 거래 회피 레짐
아래 조건에서는 기본적으로 `MODE_NO_TRADE` 또는 리스크 축소.

- 4시간 구름 안
- 1시간 Kijun 아래/위 반복 왕복
- 거래량 급감으로 유동성 부족
- 뉴스/이벤트 직후 비정상 급등락
- 상위봉 방향성과 하위봉 추세가 반복 충돌

## 24. 운영 원칙 요약

1. 전략은 수익률보다 **재현성**을 우선한다.
2. 자동매매는 진입보다 **리스크 관리와 청산**이 더 중요하다.
3. 가상머니 검증 없이 실전에 가지 않는다.
4. 백테스트가 좋더라도 페이퍼 성능이 나쁘면 실전 금지.
5. 실전에서 가장 먼저 확인할 것은 수익이 아니라 **오차와 붕괴 조건**이다.
6. 목표는 한 번의 최적화가 아니라 **검증을 통해 진화하는 시스템**이다.
