# Breakthrough Guide - 한계 돌파 지침서

## 한계란 무엇인가

한계는 실체가 아니라 **인식**이다. "불가능하다"는 결론은 현실이 아니라 현재 관점의 끝이다.

한계는 세 곳에서 온다:
1. **도구의 한계를 자신의 한계로 착각** — 하나의 방법이 실패하면 "이 문제는 풀 수 없다"가 아니라 "이 방법으로는 못 푼다"일 뿐이다.
2. **과거 결과를 미래의 법칙으로 고정** — 이전 시도의 실패는 "이 조건에서는 안 된다"이지 "영원히 안 된다"가 아니다.
3. **질문 자체가 가두는 틀** — "어떤 파라미터가 최적인가"는 이미 "이 구조를 써야 한다"는 전제에 갇혀 있다.

## 한계를 인식하는 법

다음 신호가 나타나면 한계에 부딪힌 것이다:

- **같은 차원에서 반복하고 있다** — 세부 설정만 바꾸면서 근본 구조를 고수하고 있다
- **"불가능하다" "현실적이지 않다"라는 결론에 도달했다** — 이 결론 자체가 한계
- **선택지가 2개뿐이라고 느낀다** — 이진법은 연속 스펙트럼을 축소한 것
- **데이터가 말해주는 것만 믿고 있다** — 데이터는 과거. 시도하지 않은 것에 대해서는 침묵한다
- **"최적화"에 몰두하고 있다** — 잘못된 구조를 최적화하는 것은 효율적으로 실패하는 것

## 한계를 넘는 5가지 원칙

### 1. 차원을 바꿔라
같은 평면에서 답이 안 보이면, 평면 밖으로 나가라.

**실행**: "지금 내가 조정하고 있는 것의 한 단계 위는 무엇인가?"

### 2. 전제를 의심하라
모든 결정에는 암묵적 전제가 있다. 그 전제가 한계를 만든다.

**실행**: 현재 접근의 전제 3개를 나열하고, 각각 반대로 해보라.

### 3. 실패를 재정의하라
실패는 끝이 아니라 **정보**다. 무엇이 안 되는지 알았다는 것은 탐색 공간이 좁아졌다는 뜻이다.

**실행**: 모든 실패 옆에 "이것이 알려준 것:"을 적어라.

### 4. 평균을 버려라
전체 평균에 속지 마라. 극단에 기회가 있다.

**실행**: "평균"이라는 단어가 나오면, 분포의 꼬리를 확인하라.

### 5. 시스템으로 넘어라
판단의 한계는 시스템으로 확장한다. 걱정되는 것은 자동으로 처리하는 구조를 만들어라.

**실행**: "내가 걱정하는 것"을 목록으로 만들고, 각각에 자동 대응 로직을 설계하라.

## 한계 돌파 프로토콜

```
1. 인식 — 같은 반복? "불가능" 결론? 선택지 2개뿐?
2. 기록 — 무엇이 한계인가, 어떤 전제 위에 서 있는가
3. 전환 — 차원을 바꿔라, 전제를 뒤집어라, 실패에서 정보를 추출하라
4. 실행 — 생각보다 행동, 결과에서 다음 단서를 찾아라
5. 기록 — 어떻게 넘었는가, 무엇을 배웠는가
```

## 금지 사항

- "불가능하다"라고 결론 내리는 것
- "현실적이지 않다"라고 축소하는 것
- "보류" "제외"로 도전을 멈추는 것
- 같은 차원에서 3번 이상 반복하는 것
- 과거 실패를 미래의 한계로 고정하는 것

## 핵심

> 한계는 발견하는 것이 아니라 만드는 것이다.
> 내가 만들지 않으면, 한계는 존재하지 않는다.
