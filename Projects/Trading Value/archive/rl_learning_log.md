# RL 강화학습 진행 로그

## V1 시대 (~ 2026-03-27)

### V1.0 — 이산 액션 + 수제 지표 (PPO MLP)
- **관측**: 33차원 수제 지표 (이치모쿠, 볼륨프로파일, 돈치안, 레짐 등)
- **액션**: 5개 이산 (HOLD/LONG/SHORT/CLOSE/REVERSE)
- **보상**: PnL + position_change_penalty + holding_cost + profitable_hold_bonus
- **학습**: PPO MLP, 100K~3M steps, 30분봉 기준
- **결과**: 백테스트 20전승이나 페이퍼 트레이딩 WR 38%
- **교훈**: 백테스트 성적과 실전 괴리가 큼. 복리 효과로 수치 과대 추정

### V1.1 — LSTM 도입 (RecurrentPPO)
- **변경**: MLP → LSTM (시계열 패턴 학습 기대)
- **모델**: Track C Castle (1.35M steps)
- **결과**: Sharpe 1.70, 백테스트 양호하나 실전 여전히 38% WR
- **교훈**: LSTM이 더 나았지만 관측 공간 자체가 문제

### V1.2 — 보상 구조 v2
- **변경**: Sharpe ratio 기반 보상, mark-to-market equity, volatility penalty
- **모델**: Track F Fortress, G Gladiator
- **결과**: 20분기 전승이나 페이퍼 WR 47%
- **교훈**: 보상 함수만 바꿔서는 진입 품질 개선 안 됨

### V1.3 — 시장 레짐별 파인튜닝
- **변경**: 4단계 (상승/하락/횡보/변동) 별 500K steps 순차 학습
- **결과**: 레짐별 특화되었으나 전체 성적 향상 미미
- **교훈**: 레짐 분류 자체가 수제 지표에 의존 → 근본 한계

### V1.4 — 관측 공간 확장 (35차원)
- **변경**: +VWAP deviation, +volume z-score (1분봉 집계)
- **결과**: 새 모델 from scratch → trades=0 (HOLD만 선택)
- **교훈**: 200K steps로 35차원 수렴 불가. ent_coef 부족

### V1.5 — 15분봉 primary TF + pullback 비활성화
- **변경**: 30분→15분 판단 주기, RL에 진입 직접 위임
- **결과**: 여전히 trades=0
- **교훈**: 수제 지표 35개를 모델에 주는 방식 자체가 한계

### V1 손실 분석 (sim_log 294건)

  - 부분 익절/손절, 추매, 물타기/불타기 지원

- **관측**: 305차원 (60개 15m 캔들 × 5 OHLCV + 5 포지션 상태)

- **보상**: 실현 PnL + 에쿼티 변화 (잔고 % 기준)

#### Cycle 1 결과 (2026-03-28)
- **학습**: 500K steps, ent_coef=0.05, 60일 ETH 1분봉
- **평가**: 20일 시뮬 (2025-05-25 ~ 2025-06-14)

  - 승률은 목표(65%) 초과 달성
  - 수익률은 -90.5% (과다매매로 수수료에 잠식)
  - 321건/20일 = 하루 16건 → 과다매매
  - "자주 소액 수익, 가끔 대형 손실" 패턴
- **교훈**: 승률만으로는 부족. 손익비(R:R)와 거래 빈도 제어 필요

#### Cycle 2 결과
- **학습**: 500K steps, Cycle 1 모델 이어서
- **결과**: WR 69.0%, Return -90.1%, 116거래 (80W/36L), 잔고 $988
- **개선**: 거래 수 321→116 감소
- **패턴 분석**: 풀 리버스(-1↔+1) 반복, 부분 포지션 미사용, 소액 수익+대형 손실
- **교훈**: 손익비가 핵심. 이길 때 크게, 질 때 작게

### V2.1 — 보상 구조 개선: 수익 극대화 유도 (2026-03-28)

1. **보상 이중 계산**: `_execute_trade()`에서 PnL reward + `step()`에서 equity_change로 같은 PnL 2배 계산 → 손실 거래 패널티 2배 증폭 → 거래 기피 강화. **수정 완료.**
2. **terminated/truncated 혼동**: 청산을 truncated로 처리 → value function 부정확. **수정 완료.**
3. **게이트 사후 덮어쓰기**: policy gradient 오염. **gate_overridden 시 reward=0으로 완화.**

**근본 한계 확인**:
- 330차원 관측 → 5분 방향 예측 = **51.5%** (동전 던지기)
- 어떤 RL 알고리즘도 이 예측력으로는 수수료를 이길 수 없음
- "모델이 '안 하는 게 이득'으로 수렴"은 **합리적 행동** — 에지가 없으니까
