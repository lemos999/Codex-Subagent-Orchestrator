# 코인 전략 명세서 v2

상태: `ACTIVE DRAFT`

기준 자산: `ETHUSDT Perp`, `BTCUSDT Perp`

이 문서는 [coin_strategy_spec.md](./coin_strategy_spec.md)의 아이디어를 유지하면서, 실제 자동매매 구현과 백테스트에 필요한 결정론적 규칙을 추가한 실행 스펙이다. v1 문서는 보존하고, v2를 구현 기준 문서로 사용한다.

## 1. 문서 목적

- 자동매매 전략을 사람이 임의 해석하지 않아도 되도록 기계 실행 가능한 규칙으로 정리한다.
- 실전 체결, 장애 복구, 백테스트 재현성까지 포함한 운영 기준을 명시한다.
- 전략 아이디어보다 생존성, 검증 가능성, 상태 정합성을 우선한다.

## 2. 프로젝트 목표

이 프로젝트의 목표는 "한 번 잘 맞는 전략"이 아니라, 아래 조건을 만족하는 운영 가능한 시스템을 만드는 것이다.

1. 상위 구조가 살아 있는 구간에서는 롱 편향으로 대응한다.
2. 하위 약세 정렬이 강할 때만 제한적으로 숏을 사용한다.
3. 모든 진입은 손절, 목표, 무효화 조건이 먼저 확정된 뒤에만 실행한다.
4. 백테스트, 페이퍼, 실전이 같은 규칙 집합을 사용하도록 만든다.

## 3. 범위와 비범위

### 3.1 현재 범위

- 심볼: `ETHUSDT Perp`, `BTCUSDT Perp`
- 전략 셋:
  - `TREND_LONG`
  - `PULLBACK_LONG`
  - `REBOUND_SHORT`
- 타임프레임:
  - `4h`: 상위 구조
  - `1h`: 방향 편향
  - `30m`: 실행 레짐
  - `15m`: 진입 확인
  - `5m`: 미시 트리거

### 3.2 현재 비범위

- 다중 심볼 포트폴리오 최적화
- 뉴스 해석 기반 재량 진입
- `HTF_NEUTRAL` 구간의 별도 롱 전략
- 돌파 추격 전용 전략
- 월별/레짐별 파라미터 자동 최적화

비범위 항목은 추후 검증용 실험 전략으로만 별도 추가한다.

## 4. 핵심 원칙

1. 모든 전략 판단은 완전히 마감된 봉 기준으로만 수행한다.
2. 포지션이 열려 있으면 반드시 보호 손절 주문이 존재해야 한다.
3. 같은 심볼에 대해 한 번에 하나의 순포지션만 가진다.
4. 반대 방향 포지션으로 한 번에 뒤집지 않는다.
5. `mode`와 `setup`과 `position`을 섞지 않는다.
6. 시각적 도구에 의존하는 규칙은 자동화용 고정 규칙으로 치환한다.

## 5. 데이터 입력과 지표 정의

### 5.1 필수 데이터

- OHLCV 캔들
- 거래소 서버 시각
- 주문 상태 이벤트
- 포지션 상태 이벤트
- 수수료 정보
- 슬리피지 측정용 체결 가격
- 고영향 경제 이벤트 일정

### 5.2 이치목 규칙

- 기본 파라미터는 `9 / 26 / 52 / 26 shift`
- 전략 판단용 `cloud_top`, `cloud_bottom`은 의사결정 시점의 현재 바에 정렬된 Senkou 값을 사용한다.
- 차트 UI에 보이는 forward-shift 렌더링 값을 그대로 쓰지 않는다.

### 5.3 고정 볼륨 프로파일 규칙

v1의 `Visible Range`는 자동화에 부적합하므로, 아래 고정 윈도우 프로파일로 대체한다.

| 타임프레임 | 프로파일 윈도우 |
| --- | --- |
| `30m` | 최근 96개 종가 기준 바 |
| `1h` | 최근 120개 종가 기준 바 |
| `4h` | 최근 90개 종가 기준 바 |

- `POC`, `VAH`, `VAL`은 각 타임프레임의 고정 프로파일에서 계산한다.
- `15m`, `5m`는 프로파일 기반 판단에 직접 사용하지 않는다.
- 각 프로파일은 해당 타임프레임 봉 마감 시에만 갱신한다.

### 5.4 ATR 규칙

- 모든 ATR은 `14` 기간을 사용한다.
- `ATR_5m`, `ATR_15m`, `ATR_30m`, `ATR_1h`, `ATR_4h`를 별도로 유지한다.

### 5.5 스윙 규칙

- 스윙 고점: `left=2`, `right=2` 프랙탈 기준
- 스윙 저점: `left=2`, `right=2` 프랙탈 기준
- 아직 오른쪽 2개 봉이 완성되지 않은 스윙은 확정 스윙으로 사용하지 않는다.

## 7. 정량 보조 조건

### 7.1 지지/저항 존

- `zone_width = max(current_price * 0.0015, ATR_15m * 0.25)`
- 레벨 `L`의 존은 `[L - zone_width, L + zone_width]`
- 인접 존이 겹치면 하나의 합성 존으로 병합한다.

### 7.2 상단 거절

`upper_rejection(tf, zone)`는 아래를 모두 만족할 때 `true`

1. 해당 봉의 고가가 `zone.high` 이상
2. 종가가 `zone.mid` 아래
3. 윗꼬리 길이가 몸통 길이 이상

### 7.3 하단 이탈 실패

`support_hold(tf, zone)`는 아래를 모두 만족할 때 `true`

1. 해당 봉의 저가가 `zone.low` 아래
2. 종가가 `zone.mid` 위
3. 아랫꼬리 길이가 몸통 길이 이상

### 7.4 유지

- `maintain_above(level, tf)`는 최근 연속 2개 마감봉의 종가가 `level` 위일 때
- `maintain_below(level, tf)`는 최근 연속 2개 마감봉의 종가가 `level` 아래일 때

### 7.5 박스 중앙

`box_center_30m`는 아래를 모두 만족할 때 `true`

1. 최근 48개 `30m` 봉의 최고가와 최저가 범위를 계산
2. 현재 종가가 그 범위의 `40% ~ 60%` 구간 안에 있음
3. 최근 활성 지지/저항 존과의 거리가 `0.5 * ATR_30m` 이상

### 7.6 되돌림 측정

- 기준 스윙: 가장 최근 확정 스윙 고점(`swing_high`)과 저점(`swing_low`)
- 하락폭: `drop = swing_high - swing_low`
- 되돌림 비율: `retracement_pct = (current_close - swing_low) / drop`
- `retracement_pct`가 `0.382 ~ 0.618` 범위에 있으면 "38.2%~61.8% 되돌림"으로 판정
- 기준 봉: 가장 최근 마감된 해당 타임프레임의 종가

### 7.7 비정상 변동성

`abnormal_volatility`는 아래 중 하나면 `true`

- 최근 `15m` 봉의 진폭이 `2.5 * ATR_15m` 초과
- 최근 `30m` 봉의 진폭이 `2.0 * ATR_30m` 초과

## 8. 타임프레임 상태 분류

### 8.1 4시간 상위 구조

#### `HTF_BULLISH`

아래를 모두 만족할 때

1. `4h.close > 4h.cloud_top`
2. `4h.tenkan > 4h.kijun`
3. `4h.profile_bias != below_va`

#### `HTF_BEARISH`

아래를 모두 만족할 때

1. `4h.close < 4h.cloud_bottom`
2. `4h.tenkan < 4h.kijun`
3. `4h.profile_bias == below_va`

#### `HTF_NEUTRAL`

`HTF_BULLISH`, `HTF_BEARISH` 둘 다 아니면 `HTF_NEUTRAL`

### 8.2 1시간 방향 편향

#### `H1_BULLISH`

아래를 모두 만족할 때

1. `1h.close > 1h.cloud_top`
2. `1h.tenkan >= 1h.kijun`
3. `1h.close >= 1h.kijun`
4. `1h.profile_bias != below_va`

#### `H1_BEARISH`

아래를 모두 만족할 때

1. `1h.close < 1h.cloud_bottom`
2. `1h.tenkan <= 1h.kijun`
3. `1h.close <= 1h.kijun`
4. `1h.profile_bias == below_va`

#### `H1_NEUTRAL`

`H1_BULLISH`, `H1_BEARISH` 둘 다 아니면 `H1_NEUTRAL`

### 8.3 30분 실행 레짐

#### `M30_BULLISH`

아래를 모두 만족할 때

1. `30m.close > 30m.cloud_top`
2. `30m.tenkan > 30m.kijun`
3. `30m.close >= 30m.kijun`

#### `M30_BEARISH`

아래를 모두 만족할 때

1. `30m.close < 30m.cloud_bottom`
2. `30m.tenkan < 30m.kijun`
3. `30m.close <= 30m.kijun`

#### `M30_NEUTRAL`

`M30_BULLISH`, `M30_BEARISH` 둘 다 아니면 `M30_NEUTRAL`

## 11. 전략 상세

### 11.1 `TREND_LONG`

#### 전제

1. `HTF_BULLISH`
2. `H1_BULLISH` 또는 `H1_NEUTRAL`
3. `M30_BULLISH`
4. 감시 지지 존은 `30m Tenkan`, `1h Tenkan`. 가격이 실제 접근하여 진입 트리거가 발생한 존을 "진입 기준 지지 존"이라 한다

#### 진입 순서

1. 가격이 감시 지지 존에 닿거나 존 내부로 진입
2. `5m`에서 아래꼬리 또는 직전 저점 갱신 실패 발생
3. 같은 봉 또는 다음 `5m` 봉이 이전 `5m` 고가를 종가 기준 돌파
4. 가장 최근 마감된 `15m` 봉의 종가가 진입 존의 중단 이상
5. 가장 최근 마감된 `30m` 봉의 종가가 지지 존 하단 위
6. 트리거 `5m` 거래량이 최근 5개 `5m` 평균 이상

#### 진입 가격

- 1차: 트리거 `5m` 봉 종가 기준 `50%`
- 2차: `15m` 확인 후 `30%`
- 3차: 최근 `15m` 스윙 고점 돌파 시 `20%`

#### 손절

아래 중 더 낮은 값

- 최근 확정 `5m` 스윙 저점 - `0.2 * ATR_5m`
- 사용한 지지 존 하단 - `0.2 * ATR_15m`

#### 목표가 선택 규칙

목표가는 "존재하는 저항 중 현재 진입가보다 위에 있는 값"만 유효하다.

- `tp1_candidates`: 최근 스윙 고점, `30m VAH`
- `tp2_candidates`: `1h VAH`, Fib 확장 `1.618 / 2.0 / 2.618` (기준: 가장 최근 확정 `30m` 스윙 저점 → 진입 기준 지지 존 상단. 스윙 크기가 `ATR_1h` 미만이면 Fib 확장 무효. 앵커 저점은 반드시 진입가 이하)

`tp1`은 가장 가까운 유효 저항, `tp2`는 그 다음 유효 저항으로 선택한다.  
`tp1` 기준 RR이 `1.5` 미만이면 진입하지 않는다.  
유효한 `tp2`가 없으면 `tp1` 도달 후 잔량은 트레일링만 사용한다.

#### 청산

- `tp1` 도달 시 전체 수량의 `30%`
- `tp2` 도달 시 전체 수량의 `30%`
- 잔량 `40%`는 트레일링

#### 트레일링

- `5m` 종가가 `5m Kijun` 아래 마감하면 잔량의 절반 축소
- `15m` 종가가 `15m Tenkan` 아래 마감하면 잔량 전량 청산

#### 무효화

- 사용 지지 존 하단 아래 `15m` 종가 확정
- 더 상위 지지 아래 `1h` 종가 확정
- `30m`, `15m`, `5m` 모두 `below_cloud + TK bearish`

### 11.2 `PULLBACK_LONG`

#### 전제

1. `HTF_BULLISH`
2. `H1_NEUTRAL`
3. `M30_NEUTRAL` 또는 `M30_BEARISH`
4. 핵심 지지 존은 아래 우선순위로 선택
   - `30m Kijun`
   - `1h Kijun`
   - `30m POC`
   - `30m VAL`
   - `1h POC`
   - `1h VAL`
   - `4h Kijun`

#### 진입 순서

1. 가격이 핵심 지지 존에 도달
2. 가장 최근 `15m` 봉이 `support_hold(15m, zone)`을 만족하거나, 망치형 캔들 조건을 만족
3. 다음 `5m` 마감봉 종가가 직전 3개 `5m` 고가의 최댓값을 돌파
4. 트리거 `15m` 거래량이 직전 `15m` 거래량보다 큼
5. 가장 최근 마감된 `30m` 봉 종가가 지지 존의 중단 이상

망치형 캔들은 아래를 모두 만족할 때만 인정한다.

- 몸통이 전체 캔들 길이의 `35%` 이하
- 아래꼬리가 몸통의 `2배` 이상
- 종가가 시가 이상

#### 진입 가격

- 1차: `5m` 확인 종가 기준 `50%`
- 2차: `15m` 확인 유지 시 `30%`
- 3차: `30m` 종가가 지지 존 위를 유지하면 `20%`

#### 손절

- 사용 지지 존 하단 - `0.2 * ATR_15m`
- 추가 무효화: `1h` 종가가 상위 지지 하단 아래 확정되면 전량 청산

#### 목표가 선택 규칙

진입가보다 위에 있는 값만 유효하다.

- `tp1_candidates`: `30m Tenkan`
- `tp2_candidates`: `30m Kijun`
- `tp3_candidates`: `1h Tenkan`, `1h Kijun`, `1h POC`

유효한 첫 번째 저항이 `tp1`, 그 다음이 `tp2`, 그 다음이 `tp3`가 된다.  
첫 유효 목표 기준 RR이 `1.5` 미만이면 진입하지 않는다.  
유효한 목표가 하나만 있으면 `tp1` 도달 후 잔량은 트레일링으로 관리한다.

#### 청산

- `tp1` 도달 시 `30%`
- `tp2` 도달 시 `30%`
- `tp3` 또는 트레일링 조건 도달 시 잔량 정리

#### 트레일링

- `15m` 종가가 `15m Kijun` 아래로 다시 마감하면 잔량 절반 축소
- `30m` 종가가 사용 지지 존 중단 아래로 다시 마감하면 전량 청산

#### 무효화

- 사용 지지 존 하단 아래 `15m` 종가 확정
- 선택된 상위 지지 아래 `1h` 종가 확정

### 11.3 `REBOUND_SHORT`

#### 전제

1. `M30_BEARISH` 또는 `M30_NEUTRAL + TK bearish`
2. `15m`는 `below_cloud + TK bearish`
3. `5m.close < 5m.cloud_bottom` 그리고 `5m.close <= 5m.kijun`
4. `HTF_BULLISH`에서 숏을 실행할 경우 리스크는 축소 상태만 허용

#### 감시 저항 우선순위

1. `15m Kijun`
2. `30m Tenkan`
3. `30m Kijun`
4. `30m VAH`
5. `30m POC`
6. `1h Tenkan`

#### 진입 순서

1. 선행 하락 확인: 가장 최근 확정 `30m` 스윙 고점에서 저점까지의 하락폭이 `max(ATR_30m * 1.5, current_price * 0.005)` 이상이어야 한다. 그 저점에서 하락폭의 `38.2% ~ 61.8%` 되돌림이 발생하며 감시 저항 존에 접근
2. 가장 최근 `5m` 봉이 `upper_rejection(5m, zone)`를 만족
3. 같은 봉 또는 다음 `5m` 봉이 직전 `5m` 저가를 종가 기준 하향 돌파
4. 가장 최근 `15m` 종가가 저항 존 중단 아래
5. 재하락 `5m` 거래량이 반등 `5m` 거래량보다 큼

#### 진입 가격

- 1차: 재하락 확인 `5m` 종가 기준 `50%`
- 2차: `15m` 약세 유지 확인 후 `30%`
- 3차: 직전 저점 재이탈 시 `20%`

#### 손절

- 사용 저항 존 상단 + `0.2 * ATR_15m`

#### 목표가 선택 규칙

진입가보다 아래에 있는 값만 유효하다.

- `tp1_candidates`: 직전 저점
- `tp2_candidates`: `30m POC`
- `tp3_candidates`: `30m VAL`, `1h VAL`, `4h Kijun`

첫 유효 목표 기준 RR이 `1.5` 미만이면 진입하지 않는다.  
유효한 목표가 하나만 있으면 `tp1` 도달 후 잔량은 트레일링으로 관리한다.

#### 청산

- `tp1` 도달 시 `30%`
- `tp2` 도달 시 `30%`
- 잔량 `40%`는 트레일링

#### 트레일링

- `5m` 종가가 `5m Kijun` 위로 마감하면 잔량의 `30%` 축소
- `15m` 종가가 `15m Tenkan` 위로 마감하면 잔량의 절반 축소
- `15m` 종가가 `15m Kijun` 위로 마감하면 전량 청산

#### 무효화

- 사용 저항 존 상단 위 `15m` 종가 확정
- `30m`가 다시 `above_cloud + TK bullish`
- `1h`가 `POC` 또는 `VAH` 위에서 연속 2봉 유지

## 12. 신규 진입 금지 필터

아래 중 하나면 신규 진입 금지

1. `box_center_30m == true`
2. `abnormal_volatility == true`
3. 해당 전략의 첫 유효 목표 기준 RR `< 1.5`
4. 최근 `30m` 거래량이 `volume_sma_20 * 0.6` 미만
5. 고영향 경제 이벤트 발표 `30분 전 ~ 30분 후`
6. 이미 같은 심볼에 순포지션 보유 중
7. 모든 심볼 합산 총 위험 노출이 `1.0%`에 도달
8. 일일 손실 한도, 주간 손실 한도, 연속 손실 한도 초과

## 13. 리스크 관리

### 13.1 기본 리스크

- `R` = 해당 거래의 초기 위험 금액 = `AccountBalance × Risk%`. 이후 문서에서 `0.5R`, `-3R` 등은 모두 이 단위를 기준으로 한다.

- 기본 1회 거래 리스크 상한: 총자산의 `0.35%`
- 절대 최대 1회 거래 리스크: 총자산의 `0.50%`
- 역추세 숏(`HTF_BULLISH` 구간의 `REBOUND_SHORT`): 총자산의 `0.25%` 상한
- 동시 총 위험 노출: 모든 심볼 합산 총자산의 `1.0%` 이하
- 심볼당 최대 위험 노출: 총자산의 `0.5%` 이하

### 13.2 포지션 사이징

- `TargetQty = (AccountBalance × Risk%) / |EntryPrice - StopPrice|`
- `AccountBalance`는 미실현손익을 제외한 실현잔고(Balance)
- `Risk%`는 기본 `0.35%`, 강한 확신 시 최대 `0.50%`, 역추세 숏은 `0.25%`
- `TargetQty`는 전체 포지션 목표 수량이며, 분할 진입 비율(50%/30%/20%)을 적용하여 각 단계 주문 수량 산출
- 체결 후 합산 수량이 `TargetQty`를 초과하지 않는다
- 최소 주문 단위 미만은 버린다 (ETHUSDT: 0.001 ETH, BTCUSDT: 0.001 BTC)
- 거래소 레버리지 티어에 의한 최대 수량 제한을 초과하지 않는다

### 13.3 거래 중단 규칙

- 연속 `4` 손실 이상
- 일일 손실 `-3R` 이하
- 주간 손실이 허용 한도 초과
- 주문 실패 또는 API 오류가 연속 `3`회 이상
- 슬리피지가 최근 20회 평균의 `2배` 이상 급증

### 13.4 포지션 규칙

- 분할 진입은 가능하지만 논리적으로는 하나의 집계 포지션으로 본다.
- 반대 방향 동시 보유 금지 (같은 심볼 내)
- 다른 심볼 간 반대 방향 동시 보유 허용 (예: ETHUSDT 롱 + BTCUSDT 숏)
- 손절 주문 미부착 상태로 `OPEN_*` 상태 진입 금지

### 13.5 쿨다운 규칙

- 포지션 청산 후 동일 심볼/동일 방향 재진입까지 `30m` 봉 `2`개(1시간) 대기
- 손절 청산의 경우 `30m` 봉 `4`개(2시간) 대기
- 쿨다운 중에도 반대 방향 셋업 추적은 허용

### 13.6 최대 보유 시간

- 포지션 진입 후 `30m` 봉 `48`개(24시간) 경과 시 강제 평가
- 미실현 손익이 `0.5R` 미만이면 전량 청산 (펀딩비 누적 방지)
- 미실현 손익이 `0.5R` 이상이면 트레일링 강화 (손절을 진입가 + `0.3R`로 이동)

## 14. 주문 집행 계약

### 14.1 기본 원칙

1. 진입 판단은 종가 기반, 집행은 종가 직후 주문 제출
2. 초기 진입 주문은 `marketable limit`를 사용한다.
3. 진입 주문의 최대 허용 슬리피지는 `0.05 * ATR_5m`
4. 미체결 진입 주문의 TTL은 `2개 5m 봉`
5. TTL 내 미체결이면 주문 취소 후 셋업 재평가

### 14.2 보호 손절 규칙

- 첫 체결 발생 후 즉시 보호 손절 주문을 제출한다.
- 보호 손절 주문이 거래소에서 확인되기 전에는 포지션 상태를 `OPEN_STAGE0`로 확정하지 않는다.
- 부분 체결 후 보호 손절 부착 실패 시 즉시 강제 청산 절차로 전환한다.

### 14.3 재기동 규칙

1. 저장된 상태 로드
2. 거래소 실제 포지션 조회
3. 미체결 주문 조회
4. 최근 히스토리 캔들 재수집
5. 지표 재계산
6. 상태 불일치 시 `DEGRADED` 또는 `HALTED`
7. 보호 주문 없는 실포지션 발견 시 즉시 보호 조치 후 `HALTED`

- 동일 시점 중복 신호 멱등 처리

### 16.2 수용 테스트

아래 항목은 구현 완료 전 자동 테스트 케이스로 확보한다.

1. 동일 스냅샷에서 `regime_state`는 항상 하나만 선택된다.
2. 동일 스냅샷에서 허용 전략 집합은 결정론적으로 동일해야 한다.
3. `TREND_LONG`는 미마감 `30m` 봉 정보로 진입하지 않는다.
4. `PULLBACK_LONG`는 첫 유효 목표가 진입가보다 위에 없으면 진입하지 않는다.
5. `REBOUND_SHORT`는 첫 유효 목표가 진입가보다 아래에 없으면 진입하지 않는다.
6. 첫 체결 후 보호 손절 부착 실패 시 즉시 청산 절차로 전환한다.
7. 재기동 후 로컬 상태와 거래소 상태가 다르면 거래를 중단한다.
8. `HTF_BULLISH` 구간의 숏은 기본 리스크보다 작은 값으로만 열린다.

## 17. 변경 요약: v1 -> v2

1. `Visible Range`를 고정 볼륨 프로파일 규칙으로 치환했다.
2. 정성적 표현을 가능한 범위에서 수식과 임계값으로 바꿨다.
3. 단일 `mode` 대신 `regime_state + allowed_setups` 구조로 분리했다.
4. 상세 규칙이 없던 중립 레짐 롱 경로는 현재 범위에서 제거했다.
5. 리스크 예산을 하나의 기준으로 통일했다.
6. 주문 집행, 보호 손절, 재기동 복구를 계약으로 명시했다.
7. v1의 "동일 방향 중복 포지션 최대 2개"를 "같은 심볼에 대해 하나의 순포지션"으로 변경했다.
8. 단일 심볼(`ETHUSDT`)에서 복수 심볼(`ETHUSDT + BTCUSDT`)로 범위를 확장했다.
9. 포지션 사이징 공식, 쿨다운 규칙, 최대 보유 시간을 추가했다.
10. 허용 전략 매트릭스의 빈 경로를 완전히 채웠다.
11. "급락" 등 정성적 표현을 추가 정량화했다 (스윙 기반 하락폭, 되돌림 범위).
12. Fib 확장 비율을 표준 비율(1.618/2.0/2.618)로 수정하고 앵커를 명시했다.
