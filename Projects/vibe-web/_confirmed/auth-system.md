# 사용자 인증 (Support — DEVELOPMENT-BIBLE §5 기반)

> 최종 갱신: 2026-03-24
> 근거: DEVELOPMENT-BIBLE §5 인증 & 세션 설계

## [확정] 인증 방식

NextAuth + OAuth 프로바이더

| 프로바이더 | 이유 |
|-----------|------|
| GitHub | 타겟(바이브 코더) 필수 계정 |
| Google | 비개발자 AI 아티스트 접근성 |

## [확정] 세션 규칙 (BIBLE §5)

| 규칙 | 적용 |
|------|------|
| 토큰 SHA-256 해시 후 DB 저장 | DB 유출 시 세션 탈취 방지 |
| 비밀번호 변경 시 기존 세션 무효화 | OAuth 전용이므로 해당 없음 (OAuth 토큰 갱신) |
| 계정 차단/삭제 시 즉시 반영 | JWT callback에서 매 요청 확인 |
| JWT secret 32자 이상 | NEXTAUTH_SECRET |

## [확정] 계정 삭제 처리

- 게시물/댓글 유지 + 작성자 "[탈퇴한 사용자]" 표시
- 점수/HoF 소급 취소 없음
- 정본: [common-rules.md](common-rules.md)

## [확정] 컴포넌트 인터페이스

| 연결 컴포넌트 | 입력 | 출력 | 실패 시 기본값 |
|-------------|------|------|-------------|
| 전 컴포넌트 | OAuth 인증 완료 | `userId` (CUID) | 미인증 → 로그인 페이지 리다이렉트 |
| 프로필 | `userId` | 사용자 정보 (닉네임, 아바타, OAuth 프로바이더) | — |
