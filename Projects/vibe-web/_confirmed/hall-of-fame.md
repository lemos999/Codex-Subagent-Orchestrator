# 명예의 전당 (Core 4)

> 최종 갱신: 2026-03-24
> 주관 섹션: 명예의 전당
> 참조 섹션: 갤러리(Core 1), 커뮤니티 게시판(Core 5), 공통 규칙

## [확정] 선정 기준

`likeCount` 단독 — `viewCount` 미반영. 대상: 갤러리 작품 + 게시판 게시글 (공용 `postId`로 식별)

| 구분 | 선정 방식 | 대상 수 | 동점 처리 |
|------|----------|---------|----------|
| 주간 베스트 | likeCount 내림차순 Top 5 (자동) | 5건 | 최신 게시물 우선 |
| 월간 명예의 전당 | likeCount 내림차순 Top 10 (자동) | 10건 | 최신 게시물 우선 |

## [확정] likeCount 합산 범위

갤러리 + 게시판 전체 `likeCount` 합산 (창작 활동 전체 인정)

## [확정] 갱신 주기

| 구분 | 갱신 시점 | 실행 방식 |
|------|----------|----------|
| 주간 베스트 | 매주 월요일 | 독립 실행 |
| 월간 명예의 전당 | 매월 1일 | 독립 실행 |

> 주간과 월간은 상호 의존 없음 — 각각 독립 실행

## [확정] 활성화 조건

활성 유저 10명 미만 시 HoF 비활성화 (소수 독점 방지)
활성 유저 정의: 최근 7일 내 1회 이상 활동 (업로드/댓글/좋아요 중 하나)

## [확정] likeCount=0 처리

| 상황 | 표시 방식 |
|------|----------|
| 해당 주/월 likeCount=0 | 이전 주/월 데이터 유지 + "지난 주/월 기준" 라벨 표시 |
| 서비스 첫 주/월 (이전 데이터 없음) | "데이터 집계 중" 표시 |

## [확정] 동점 처리 규칙

동점 시 최신 게시물 우선 (게시물 생성 시각 기준)
수치 정본: [common-rules.md](common-rules.md)

## [확정] 컴포넌트 인터페이스

| 연결 컴포넌트 | 입력 (값, 범위) | 출력 (값, 범위) | 실패 시 기본값 |
|-------------|---------------|---------------|-------------|
| 갤러리(Core 1) + 게시판(Core 5) | `likeCount` (전체 합산) | — | 0 |
| 홈 페이지 | — | 주간 베스트 위젯 + 월간 명예의 전당 위젯 | 이전 주/월 데이터 또는 "데이터 집계 중" |

## [확정] 모순 검증 통과 항목

| 모순 | 해결 |
|------|------|
| likeCount=0 명예의 전당 처리 | 이전 주 유지 + 라벨 표시 (2:1 합의) |
| 명예의 전당 viewCount 반영 여부 | likeCount 단독 (2:1 합의) |
| 주간+월간 갱신 동시 처리 | 독립 실행 (만장일치) |
| likeCount 합산 범위 | 갤러리+게시판 전체 합산 (만장일치) |
