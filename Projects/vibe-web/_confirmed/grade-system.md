# 사용자 등급 시스템 (Core 3)

> 최종 갱신: 2026-03-24
> 주관 섹션: 등급 시스템
> 참조 섹션: 갤러리(Core 1), 커뮤니티 게시판(Core 5), 명예의 전당(Core 4)

## [확정] 등급 구조

6단계 + 단계별 5레벨 = 총 Lv.30

| 단계 | 진입 `expPoints` | Lv.1 | Lv.2 | Lv.3 | Lv.4 | Lv.5 |
|------|----------------|------|------|------|------|------|
| 뉴비 | 0 | 0 | 10 | 20 | 30 | 40 |
| 루키 | 50 | 50 | 70 | 90 | 110 | 130 |
| 크리에이터 | 150 | 150 | 210 | 270 | 330 | 390 |
| 프로 | 400 | 400 | 480 | 560 | 640 | 720 |
| 마스터 | 800 | 800 | 940 | 1080 | 1220 | 1360 |
| 레전드 | 1500 | 1500~ (상한 없음, 명예 포인트로 계속 누적) | | | |

> 뉴비 수치 (0~49, 균등 5분할): Lv.1=0, Lv.2=10, Lv.3=20, Lv.4=30, Lv.5=40
> 루키 수치 (50~149, 균등 5분할): Lv.1=50, Lv.2=70, Lv.3=90, Lv.4=110, Lv.5=130 → 150에서 크리에이터 승급
> 크리에이터 수치 (150~399, 균등 5분할): Lv.1=150, Lv.2=210, Lv.3=270, Lv.4=330, Lv.5=390 → 400에서 프로 승급
> 프로 수치 (400~799, 균등 5분할): Lv.1=400, Lv.2=480, Lv.3=560, Lv.4=640, Lv.5=720 → 800에서 마스터 승급
> 마스터 수치 (800~1499, 균등 5분할): Lv.1=800, Lv.2=940, Lv.3=1080, Lv.4=1220, Lv.5=1360 → 1500에서 레전드 승급

## [확정] activityScore 계산 규칙

점수 상세는 [common-rules.md](common-rules.md) 참조. 아래는 요약:

| 행동 | 점수 | 제한 | 정본 |
|------|------|------|------|
| 업로드 | +10pt | 일일 30pt | common-rules.md |
| 받은 좋아요 | +2pt | 자기 좋아요 제외 | common-rules.md |
| 댓글 작성 | +2pt | 일일 20pt | common-rules.md |
| 받은 댓글 | +1pt | 동일 게시물 1댓글만 | common-rules.md |
| viewCount | 0pt | 미반영 | common-rules.md |

## [확정] 자기 좋아요 금지 규칙

자기 게시물에 좋아요 불가 — `likeReceived` 미집계

## [확정] 삭제 및 쿨다운 규칙

- 게시물/댓글 삭제 시 점수 차감 없음
- 삭제 후 24시간 재업로드 1건 제한 (갤러리 전체 기준, 카테고리 무관)

## [확정] 레전드 이후 처리

- `expPoints` 계속 누적 (Lv.30 이후에도 중단 없음)
- 별도 "명예 포인트" 트랙 표시 전용 (등급 영향 없음)
- 프레스티지 시스템: v2 검토 (MAU 1,000+ 이후)

## [확정] 컴포넌트 인터페이스

| 연결 컴포넌트 | 입력 (값, 범위) | 출력 (값, 범위) | 실패 시 기본값 |
|-------------|---------------|---------------|-------------|
| 갤러리(Core 1) | `uploadCount` (갤러리 작품 전용) | `activityScore` 업데이트 | 스킵 |
| 게시판(Core 5) | `commentCount` (전체 합산) | `activityScore` 업데이트 | 스킵 |
| 갤러리+게시판 | `likeReceived` (전체 합산) | `activityScore` 업데이트 | 스킵 |
| 갤러리+게시판 | `commentReceived` (전체 합산) | `activityScore` 업데이트 | 스킵 |
| 프로필 | — | `userLevel` 출력 | — |
| 명예의 전당(Core 4) | — | `userLevel` (등급 기반 필터) | — |

## [확정] 모순 검증 통과 항목

| 모순 | 해결 |
|------|------|
| 레전드 이후 expPoints 처리 | 계속 누적 (2:1 합의) |
| viewCount → activityScore 포함 여부 | 제거 (만장일치) |
