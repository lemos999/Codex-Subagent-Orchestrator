# Phase 3 확정 항목 — vibe-web

## Charter (Phase 1)
- **Primary Outcome:** AI 콘텐츠를 자랑하고 싶은 욕구 충족. 핵심 시나리오: 작품 업로드 → 피드백 → 등급 상승 → 명예의 전당
- **Operating Loop:** 마이크로(브라우징→클릭→좋아요) / 미들(업로드→피드백→등급) / 매크로(포트폴리오→명예의전당)
- **Differentiation:** AI 제작 과정(프롬프트, 도구, 워크플로우) 공유 → 재현 가능한 영감
- **Target:** 바이브 코더/AI 아티스트, 데스크톱+모바일, 매일 방문, 낮은 복잡도
- **Scope:** 1인 개발, 린, MVP

## Core 1: 갤러리 시스템
- [필수] 레이아웃: 메이슨리 (Pinterest식) — 카드 높이 가변, 콘텐츠 타입별 자연스러운 배치
- [필수] 카드 정보: 리치 — 썸네일 + 제목 + 작성자 + 좋아요 + 도구 태그 + 제작과정 미리보기 1줄

## Core 2: 작품 상세 + 제작과정
- [필수] 상세 페이지: 원스크롤 — 작품 → 제작과정 → 댓글 세로 배치
- [필수] 제작과정 입력: 자유 텍스트 (마크다운 에디터)
- 입출력 계약: 갤러리 카드 클릭 → postId / viewCount +1 (표시용, activityScore 미반영)

## Core 3: 사용자 등급 시스템
- [필수] 등급 구조: 6단계 + 단계별 5레벨 (총 Lv.30)
  - 뉴비(0) → 루키(50) → 크리에이터(150) → 프로(400) → 마스터(800) → 레전드(1500)
  - 각 단계 내 Lv.1~5, 균등 분배 (예: 루키 Lv.1=50, Lv.2=70, Lv.3=90, Lv.4=110, Lv.5=130 → 150에서 크리에이터 승급)
- [필수] 승급 기준: 복합 점수
  - 업로드 (+10) + 받은 좋아요 (+2) + 댓글 작성 (+2, 일일 상한 20pt) + 받은 댓글 (+1)
  - viewCount는 activityScore에 포함하지 않음 (조작 방지)
  - 자기 좋아요 금지 (자기 게시물 likeCount 미집계)
  - 동일 게시물 1댓글만 점수 인정
  - 삭제 시 점수 차감 없음, 단 삭제 후 24h 쿨다운 (같은 카테고리 재업로드 제한)
- [필수] 레전드 이후: 별도 "명예 포인트" 누적 트랙 (표시 전용, 등급 영향 없음). 프레스티지는 v2 검토
- 입출력 계약:
  - 입력: 갤러리 → uploadCount (갤러리 작품 전용), 게시판+갤러리 → commentCount (전체 합산), 좋아요 → likeReceived (갤러리+게시판 합산), commentReceived (갤러리+게시판 합산)
  - 출력: 프로필 → userLevel, 명예의 전당 → 등급 기반 필터, expPoints → 레벨 산정

## Core 4: 명예의 전당
- [필수] 선정 기준: likeCount 단독 Top 5(주간) / Top 10(월간) 자동 선정 (viewCount 미반영)
- [필수] 활성화 조건: 활성 유저 10명 미만 시 HoF 비활성화 (소수 독점 방지)
- [필수] 갱신 주기: 주간 베스트 (매주 월요일) + 월간 명예의 전당 (매월 1일) — 독립 실행 (상호 의존 없음)
- [필수] likeCount=0 처리: 이전 주/월 데이터 유지 + "지난 주/월 기준" 라벨 표시. 서비스 첫 주/월(이전 데이터 없음)은 "데이터 집계 중" 표시
- [필수] 좋아요 합산: 갤러리 + 게시판 전체 likeCount 합산 (창작 활동 전체 인정)
- 입출력 계약:
  - 입력: 갤러리+게시판 → likeCount (전체 합산)
  - 출력: 홈 페이지 → 주간 베스트 위젯 + 월간 명예의 전당 위젯

## Core 5: 커뮤니티 게시판
- [필수] 카테고리: 5개 — 자유 / 질문 / 팁 공유 / 프로젝트 모집 / 공지
- 입출력 계약:
  - 입력: userId, categoryId
  - 출력: 등급 시스템 → commentCount, likeReceived → activityScore

## 공통 규칙
- [필수] 계정 삭제: 게시물/댓글 유지 + 작성자 "[탈퇴한 사용자]" 표시. 점수/HoF 소급 취소 없음
- [필수] 좋아요 유일성: 계정당 게시물 1좋아요만 허용. 취소 시 likeCount -1
- [필수] 업로드 점수 일일 상한: 30pt (3건/일). 대량 업로드 파밍 방지
- [필수] 신고/제재: 존재 + 최소 카테고리 4개 (스팸 / 욕설 / 저작권 침해 / 기타). 제재 절차는 구현자 재량
- [필수] HoF 동점 규칙: 동점 시 최신 게시물 우선

## Support (린 스코프 — 필수만)
- 사용자 인증 → userId (전 컴포넌트)
- 프로필/포트폴리오 → userLevel 표시
- 카테고리/태그 → 탐색 보조
- 검색 → 기본 텍스트 검색
- 반응형 레이아웃 → 데스크톱+모바일
