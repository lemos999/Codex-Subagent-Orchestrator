# Readiness Report 형식

> Phase 5에서 생성하는 문서의 표준 형식.

---

## 1. CLAUDE.md (프로젝트 지침)

```markdown
# [프로젝트명] — 프로젝트 지침

## 역할
당신은 [도메인]의 설계 디렉터입니다.
핵심 가치가 모든 판단의 최우선입니다.

## 설계 방향
- [공통 원칙 참조: design-director SKILL.md]
- [프로젝트 특화 원칙]

## 검증 프로토콜
검증 4단계(오류→모순→맥락→실행)를 따르되, 3단계(맥락) 파라미터:
- [도메인 컨텍스트 축1]: ___
- [도메인 컨텍스트 축2]: ___

## 섹션 분류
| 섹션 | 담당 컴포넌트 |
|------|-------------|

## Charter 요약
- Primary Outcome: ___
- Operating Loop: 마이크로/미들/매크로
- Differentiation: ___

## 운용 규칙
- `_confirmed/`에 쓸 수 있는 건 검증 통과 항목뿐
- 태그: 확정 / 미결 / 보류(기한 필수)
- 미결/보류는 `_confirmed/` 진입 불가
```

## 2. _confirmed/ 파일 형식

```markdown
# [컴포넌트명]

> 최종 갱신: [YYYY-MM-DD]
> 주관 섹션: [섹션명]
> 참조 섹션: [섹션명1, 섹션명2]

## [확정] [항목명]
[수치 또는 규칙 또는 조건]

## [확정] [항목명] — Step-by-Step
| 단계 | 입력 | 처리 | 출력 |
|------|------|------|------|
| Step 1 | [입력] | [처리] | [출력] |

## [확정] [수치 테이블]
| 항목 | 수치 | 비고 |
|------|------|------|

## [확정] 컴포넌트 인터페이스
| 연결 컴포넌트 | 입력 (값, 범위) | 출력 (값, 범위) | 실패 시 기본값 |
|-------------|---------------|---------------|-------------|

## [확정] 모순 검증 통과 항목
| 모순 | 해결 |
|------|------|

## [보류] [항목명] (기한: YYYY-MM-DD)
방향: ___
```

## 7. sections/ 파일 형식

```markdown
# [섹션명] 기획서

> 최종 갱신: [YYYY-MM-DD]

## 담당 컴포넌트
- [컴포넌트명1] (확정: → _confirmed/[컴포넌트명1].md)
- [컴포넌트명2] (확정: → _confirmed/[컴포넌트명2].md)

## 미결 사항
| 항목 | 상태 | 의존 | 기한 |
|------|------|------|------|

## 설계 노트
[_confirmed/에 들어가지 않는 설계 의도, 대안 비교, 참고 자료 등]
```

## 태그 범례

| 태그 | 이모지 별칭 | 의미 | `_confirmed/` 진입 |
|------|-----------|------|-------------------|
| 확정 | O | 검증 완료 | 가능 |
| 미결 | ? | 미확정 | **불가** |
| 보류 | ~ | 방향 확정, 수치 미정 | **불가** (기한 필수) |
| [필수] | — | 실행 전 확정 대상 | Phase 3 인터뷰 대상 |
| [준필수] | — | 1~2주 내 확정 대상 | Phase 3 인터뷰 대상 |
| [선택] | — | 실행 중 확정 가능 | 선택적 |
