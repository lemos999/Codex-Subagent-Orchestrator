# State Model — 공통 상태 모델

> 프로젝트 전체에서 사용하는 상태/태그/흐름의 통합 정의.

---

## 항목 상태 태그

| 태그 | 이모지 별칭 | 의미 | `_confirmed/` 진입 |
|------|-----------|------|-------------------|
| 확정 | O | 검증 완료, 실행 가능 | 가능 |
| 미결 | ? | 미확정, 추가 결정 필요 | **불가** |
| 보류 | ~ | 방향 확정, 수치 미정 (기한 명시 필수) | **불가** |

도메인 팩이 별도 이모지 표기를 정의하면 해당 표기를 우선 사용.

## 우선순위 태그

| 태그 | 산출물 표기 | 의미 | 확정 시점 |
|------|-----------|------|----------|
| 필수 | [필수] | 실행 전 반드시 확정 | Phase 3 Decision Card |
| 준필수 | [준필수] | 실행 후 1~2주 내 확정 | Phase 3 또는 실행 초기 |
| 선택 | [선택] | 실행 중 확정 가능 | 자유 |

산출물 파일에서 우선순위 태그는 항목 제목 앞에 `[필수]`, `[준필수]`, `[선택]`으로 표기.

---

## 태그 전환 규칙

```
미결 ──[Decision Card 확정 + Phase 4 검증]──> 확정
보류 ──[수치 확정 + Phase 4 검증 (기한 내)]──> 확정
확정 ──[Phase 6 변경 요청]──> 미결 (재검증 완료까지)
보류 ──[기한 초과]──> 미결 → Phase 3 복귀
```

### 전환 조건 상세

| 전환 | 트리거 | 필수 조건 |
|------|--------|----------|
| 미결 -> 확정 | Decision Card 인터뷰 완료 | Phase 4 검증 4단계 통과 |
| 보류 -> 확정 | 수치 확정 | Phase 4 검증 통과 + 기한 내 |
| 확정 -> 미결 | Phase 6 변경 요청 접수 | 재검증 완료까지 일시 전환 |
| 보류 -> 미결 | 보류 기한 초과 | Phase 3 복귀. Core 연결 항목이면 "실행 착수 가능" 선언 보류 |

## Phase 상태 흐름

```
[Phase 0: Intake] ─gate─> [Phase 1: Charter] ─gate─> [Phase 2: Map]
       ↓                                                    ↓
  도메인 확정                                          컴포넌트 분류
                                                           ↓
[Phase 3: Decide] ─gate─> [Phase 3.5: Integrate] ─gate─> [Phase 4: Verify]
       ↓                                                    ↓
  Decision Card                                      "실행 착수 가능"
                                                           ↓
                                                  [Phase 5: Package]
                                                           ↓
                                                  [Phase 6: Change] (필요 시)
```

각 Phase 게이트 미통과 시: 다음 Phase 진입 **금지**.
FAIL 발생 시: 해당 컴포넌트의 관련 Phase로 복귀.

---

## 단일 진실 소스 원칙

- 하나의 수치/규칙은 하나의 `_confirmed/` 파일에만 정의
- 다른 파일에서 동일 수치 필요 시 참조(링크)로 연결
- 수정은 정본 위치에서만, 참조는 자동 반영
