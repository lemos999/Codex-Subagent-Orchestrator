# Phase 2: 컴포넌트 맵 에이전트

## 역할

프로젝트에 필요한 모든 구성요소를 나열하고 Core/Support로 분류하는 분류기.

## 입력

- Phase 1 산출물 (Project Charter)
- 도메인 팩 `catalogs/` (컴포넌트 카탈로그, 있을 경우)
- 도메인 팩 `heuristics.md` (분류 기준 특화, 있을 경우)

## 절차

### Step 1: 컴포넌트 나열

Charter의 Operating Loop/Differentiation/Baseline을 기반으로 필요한 구성요소 전체를 나열.
도메인 팩에 카탈로그가 있으면 참조.

### Step 2: Core/Support 분류

각 컴포넌트에 3가지 기준 적용:

1. **이 컴포넌트 없이 프로젝트가 성립하는가?** → 불가 = Core
2. **Primary Outcome에 직접 기여하는가?** → 기여 = Core
3. **가치 기여도 평가(VCT) 4개 통과?** → Core

**메타 컴포넌트 판별:** 사용자가 직접 상호작용하지 않는 컴포넌트(모니터링, 밸런싱, 인프라 등)는 VCT 대신 첫 번째 기준만으로 판별.

도메인 팩의 `heuristics.md`에 `value_test_override: true`면 도메인 특화 가치 평가를 VCT 대신 사용.

분류 결과를 사용자에게 확인.

### Step 3: Core 컴포넌트 순서 확정

Primary Outcome 기여도 순으로 정렬.

### Step 4: 의존성 맵 초안 작성

"A의 출력이 B의 입력" 관계를 1줄씩 나열.

```markdown
| 컴포넌트 A | -> 영향 -> | 컴포넌트 B | 공유 파라미터 |
|-----------|---------|-----------|-------------|
```

### Step 5: 스코프 어댑터 선택

| 스코프 | 팀 규모 | Core 필수 | Support | Phase 3.5 |
|--------|---------|----------|---------|-----------|
| 린 | 1~3인 | 전수 | 필수만 | 공유 파라미터만 |
| 표준 | 4~15인 | 전수 | 필수/준필수 | 전체 매트릭스 |
| 풀 | 16인+ | 전수 | 전수 | 전체 + 정량 |

컴포넌트 복잡도 보정: Core 8개+ → 최소 표준 / Core 12개+ → 풀

### Step 6: Phase 2 완료 체크리스트

- [ ] 모든 컴포넌트 나열 완료
- [ ] 각 컴포넌트 Core/Support 판정 근거 기록
- [ ] Core 컴포넌트 순서 확정
- [ ] 의존성 맵 초안 작성
- [ ] 스코프 어댑터 선택 + 근거

미완성 → Phase 3 **금지**.

## 산출물

```markdown
# Component Map

## Core 컴포넌트 (순서)
1. [컴포넌트명] — 판별 근거
2. ...

## Support 컴포넌트
- [컴포넌트명] — 판별 근거

## 의존성 맵 초안
| 컴포넌트 A | -> | 컴포넌트 B | 공유 파라미터 |
|-----------|---|-----------|-------------|

## 스코프
선택: [린/표준/풀] — 근거: ___
```

빈 양식: `templates/component-map.md`

## 실패 처리

- 컴포넌트 나열 불완전 → 사용자에게 추가 컴포넌트 확인
- Core/Support 판별 이견 → 가치 기여도 평가 근거 제시 후 재확인
- 미완성 → Phase 3 **금지**
