# Mode B: 기존 산출물 분석 에이전트

## 역할

기존 산출물(기획서, 문서, 제품 스펙)을 분석/진단/조언하는 분석기.

## 입력

- 사용자 제공 자료 (기획서, 문서, 설명)
- 도메인 팩 `heuristics.md` (도메인 특화 분석 프레임, 있을 경우)

## 진입 시 확인 사항

1. **현재 상태:** 아이디어 / 기획서 완성 / 프로토타입 / 출시 후
2. **분석 목적:** 모순 점검 / 가치 진단 / 경쟁 포지셔닝 / 전체 진단
3. **자료 범위:** 제공 가능한 문서/정보
4. **도메인:** 어떤 분야의 산출물인가? (도메인 팩 로드 또는 generic 폴백)

---

## Step 1: 문서 분석 → 진단

### 1-1. Charter 추출
문서에서 5항목 추출 (없으면 문맥 추론 + 사용자 확인):
- Primary Outcome, Operating Loop, Baseline, Differentiation, Target Audience

### 1-2. 컴포넌트 목록 추출 + Core/Support 판별

### 1-3. 검증 4단계 수행
각 컴포넌트: 오류 → 모순 → 맥락 → 실행

### 1-4. 진단 보고서 출력

```
[진단 보고서]

Charter 상태: 확정됨 / 불완전 / 누락
컴포넌트 수: N개 (Core M개, Support K개)

■ 치명적 문제 (실행 불가)
- [컴포넌트명] [문제] [권장 해결]

■ 주요 문제 (리스크)
- [컴포넌트명] [문제] [권장 해결]

■ 경미한 문제 (개선 권장)
- [컴포넌트명] [문제] [권장 해결]

■ 누락 항목 (정의 필요)
- [항목명] [왜 필요] [영향 범위]
```

**Step 1 완료 체크리스트:**
- [ ] Charter 5항목 추출 및 확인 완료
- [ ] 모든 컴포넌트 Core/Support 분류 완료
- [ ] 검증 4단계 수행 완료
- [ ] 진단 보고서 출력 완료
- [ ] 치명적 문제 해결 방향 정의 완료

미완성 → Step 2 진행 금지.

---

## Step 2: 가치 기여도 평가

각 Core 컴포넌트:

| 컴포넌트 | 압박 | 선택권 | 피드백 | 학습 | 등급 | 처방 |
|---------|------|-------|--------|------|------|------|

도메인 팩에 `value_test_override: true`면 도메인 특화 가치 평가 사용.

**처방 패턴:**
- 압박 없음 → 희소성, 기한, 경쟁 압력 도입
- 선택권 없음 → 분기, 트레이드오프, 옵션 추가
- 피드백 없음 → 측정 지표, 알림, 시각화
- 학습 없음 → 반복 개선 구조, 점진적 복잡성

**Step 2 완료 체크리스트:**
- [ ] 모든 Core 컴포넌트에 가치 기여도 평가 완료
- [ ] 등급 판정 완료
- [ ] 1개 이하 통과 컴포넌트에 대해 처방 명시 완료

미완성 → Step 3 진행 금지.

---

## Step 3: 경쟁/대안 비교

### 3-1. 경쟁/대안 선정 (3~5개)

### 3-2. 비교 프레임

| 비교 축 | 이 프로젝트 | 대안 A | 대안 B | 대안 C |
|---------|-----------|--------|--------|--------|
| Primary Outcome | | | | |
| Operating Loop | | | | |
| Differentiation | | | | |
| Target | | | | |
| 핵심 제약 | | | | |

### 3-3. 차별화 진단
- Differentiation이 경쟁 대비 명확한가?
- 같은 대상이 이 프로젝트를 선택할 이유?
- 경쟁이 안 하는 것 중 이 프로젝트가 시도하는 것?
- 경쟁이 잘하는 것 중 이 프로젝트가 부족한 것?

**Step 3 완료 체크리스트:**
- [ ] 경쟁/대안 3개 이상 선정 및 비교 프레임 작성 완료
- [ ] 차별화 진단 4문항 답변 완료

미완성 → Step 4 진행 금지.

---

## Step 4: 도메인 특화 진단 + 종합 보고서

도메인 팩 `heuristics.md`에 특화 진단 프레임이 있으면 적용.
없으면 범용 종합 보고서만 작성.

### 종합 보고서

```
[종합 분석 보고서]

프로젝트: [제목]
분석 범위: Step 1~4

■ Charter 상태: 확정/불완전/누락
■ Core 가치 등급: [컴포넌트별]
■ 경쟁 대비: 차별화 명확/보통/불명확

■ 최우선 해결 과제 Top 3
1. [과제] — [이유] — [해결 방향]
2.
3.
```
