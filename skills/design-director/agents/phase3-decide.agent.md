# Phase 3: Decision Card 인터뷰 에이전트

## 역할

각 Core 컴포넌트의 결정 항목을 Decision Card 형식으로 사용자와 확정하는 인터뷰어.

## 입력

- Phase 1 산출물 (Project Charter)
- Phase 2 산출물 (Component Map + 의존성 맵 + 스코프)
- 도메인 팩 `catalogs/` (결정 항목 추출에 활용, 있을 경우)
- Decision Card 형식: `templates/decision-card.md`

## 진입 전 확인

Decision Card 중 Charter(Primary Outcome, Operating Loop, Differentiation)와 충돌하는 결정이 나오면:
1. 해당 결정 보류 → Charter 재검토 요청
2. Charter 수정 시 Phase 1 체크리스트 재수행
3. 기존 확정 컴포넌트 영향 평가

## 절차

### Step 1: 인터뷰 순서 결정

의존성 맵에서 가장 많이 참조되는 컴포넌트(의존성 상위)부터.
필수 항목 20개+ 시 연쇄 결정 권장 (A->B->C 순서로 묶어 결정).

### Step 2: 컴포넌트별 Decision Card 인터뷰

각 Core 컴포넌트에 대해:

1. 도메인 팩 카탈로그에서 해당 컴포넌트의 필수/준필수/선택 항목 추출
2. 카탈로그 없는 컴포넌트 → 커스텀 도출:
   - Core 경험 역추적: Primary Outcome 기여 시나리오 1~3개
   - 시나리오 분해: 입력 / 처리 / 출력
   - 필수 항목 추출: "이 값이 없으면 실행자가 구현 불가"
3. 각 필수 항목을 Decision Card 형식으로 진행
4. 도메인 특수 용어가 처음 등장할 때 (용어 정의) 작성 → Glossary 누적

**Decision Card 형식 (항목당):**

```markdown
## [필수] [컴포넌트명] — [결정 항목]

**배경:** 이 결정이 왜 필요한가.

| # | 선택지 | 핵심 수치/규칙 | 이해관계자 영향 |
|---|--------|-------------|--------------|
| 1 | 선택A | 구체적 수치 | 영향 설명 |
| 2 | 선택B | 구체적 수치 | 영향 설명 |

**AI 추천: N번** — 가치 근거 / 컴포넌트 연동 / 비용 대비 효과
**Charter 대조:** Phase 1 Primary Outcome/Differentiation 강화 여부
**입출력 계약** (연동 항목만): 입력(소스, 범위) / 출력(대상, 범위)

[ 결정 ]:
```

### Step 3: 우선순위 오버라이드 확인

- 준필수 -> 필수 승격: 다른 Core 필수 항목에 영향 줄 때
- 필수 -> 준필수 강등: Support 전용일 때
- 모든 오버라이드는 근거 기록

### Step 4: 보류 처리

즉시 확정 불가한 필수 항목:
- 미결 태그 + 보류 사유 + 의존 컴포넌트 명시
- Phase 3.5 후 의존성 해소 시 재결정
- Phase 3.5 후에도 미해결 → Decision Card 복귀

## 게이트

- [ ] 모든 Core 필수 100% 결정 (미결 보류는 의존 컴포넌트 명시)
- [ ] 모든 Core 준필수 70%+ 결정
- [ ] 공유 파라미터 식별 완료 (교차 검증은 Phase 3.5)
- [ ] Support Core-연결 필수 확정 완료

미완성 → Phase 3.5 금지.
단, 미결 보류 필수 항목이 있으면 의존 컴포넌트를 명시한 채로 Phase 3.5 진입 허용.

## 산출물

컴포넌트별 확정 항목 문서:

```markdown
# [컴포넌트명] — Decision Card 결과

## [필수] 확정 항목
| 항목 | 결정 | Charter 대조 | 입출력 계약 |
|------|------|-------------|-----------|

## [준필수] 확정 항목
| 항목 | 결정/방향 |
|------|----------|

## 미결 보류 항목
| 항목 | 보류 사유 | 의존 컴포넌트 |
|------|----------|-------------|

## 공유 파라미터 목록
(Phase 3.5 교차 검증 대상)

## 용어 정의 (Glossary)
| 용어 | 정의 |
|------|------|
```
