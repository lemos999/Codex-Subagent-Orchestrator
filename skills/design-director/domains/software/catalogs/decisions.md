# Software Decision Reference

> Phase 3 Decision Card 인터뷰 시 참조하는 공통 결정 항목 레퍼런스.
> 각 항목의 기본 태그는 참고용 — 프로젝트 맥락에 따라 조정 가능.

## API / 인터페이스

| 결정 항목 | 질문 | 선택지 예시 | 기본 태그 |
|----------|------|-----------|----------|
| 통신 프로토콜 | 클라이언트-서버 간 주요 통신 방식은? | REST, gRPC, GraphQL, WebSocket | 필수 |
| API 버전 전략 | 하위 호환 깨질 때 어떻게 관리? | URI 경로, 헤더, 쿼리 파라미터, 무버전 | 필수 |
| 에러 응답 형식 | 클라이언트에 에러를 어떤 구조로 전달? | RFC 7807, 커스텀 envelope, HTTP 상태만 | 필수 |
| 인증 방식 | API 호출자 신원 확인 방법은? | JWT, OAuth 2.0, API Key, mTLS | 필수 |
| Rate limiting | 과도한 요청 어떻게 제한? | 고정 윈도우, 슬라이딩 윈도우, 토큰 버킷 | 준필수 |
| 페이지네이션 | 대량 목록 조회 시 분할 방식은? | 오프셋, 커서, 키셋 | 준필수 |
| 요청/응답 형식 | 직렬화 포맷은? | JSON, Protocol Buffers, MessagePack | 선택 |

## 데이터

| 결정 항목 | 질문 | 선택지 예시 | 기본 태그 |
|----------|------|-----------|----------|
| 주 저장소 유형 | 핵심 데이터를 어디에 저장? | RDB, Document DB, KV Store, 그래프 DB | 필수 |
| 정규화 수준 | 데이터 중복 vs 조회 성능 균형은? | 3NF, 비정규화, 혼합 (CQRS read model) | 필수 |
| 마이그레이션 전략 | 스키마 변경을 어떻게 배포? | 선행 마이그레이션, 후행 마이그레이션, expand-contract | 필수 |
| 캐시 무효화 | 캐시 데이터 갱신 시점은? | TTL, 이벤트 기반, 수동, write-through | 준필수 |
| 백업/복구 | RPO·RTO 목표는? | RPO: 1분/1시간/1일, RTO: 5분/1시간/4시간 | 준필수 |
| 데이터 보존 정책 | 오래된 데이터 처리는? | 아카이브, 삭제, 영구 보존 | 선택 |

## 비즈니스 로직

|----------|------|-----------|----------|
| 트랜잭션 경계 | 여러 연산의 원자성 보장 범위는? | 단일 DB 트랜잭션, 사가 패턴, 보상 트랜잭션 | 필수 |
| 동시성 모델 | 동시 수정 충돌 해결 방식은? | 낙관적 잠금, 비관적 잠금, CRDT, 최종 기록 우선 | 필수 |
| 멱등성 보장 | 중복 요청 시 안전한 처리 방법은? | 멱등성 키, 자연 멱등성, 중복 감지 | 준필수 |
| 도메인 이벤트 | 상태 변경을 외부에 어떻게 알리는가? | 이벤트 발행, 폴링, 변경 데이터 캡처 | 준필수 |
| 장기 실행 프로세스 | 수 분~수 일 걸리는 작업 관리는? | 상태 머신, 워크플로 엔진, 비동기 작업 큐 | 선택 |

## 인증 / 보안

| 감사 로그 범위 | 어떤 이벤트를 기록? | 인증 시도, 데이터 변경, 관리자 작업 | 준필수 |
| 입력 검증 | 사용자 입력 검증 위치/방식은? | API 계층, 도메인 계층, 양쪽 | 준필수 |

## 인프라 / 운영

| 결정 항목 | 질문 | 선택지 예시 | 기본 태그 |
|----------|------|-----------|----------|
| 배포 전략 | 새 버전을 어떻게 릴리스? | 블루-그린, 카나리, 롤링, 재생성 | 필수 |
| 장애 감지 | 서비스 이상을 어떻게 탐지? | 헬스체크, 서킷브레이커, 이상 탐지 | 필수 |
| 스케일링 전략 | 부하 증가 시 어떻게 대응? | 수평 확장, 수직 확장, 자동 스케일링 | 준필수 |
| 롤백 기준 | 배포 실패 판단과 복구 방법은? | 에러율 임계, 지연 임계, 수동 판단 | 준필수 |
| 로그 수준 정책 | 어떤 로그를 어디까지 남기는가? | 구조화 로그, 샘플링, 보존 기간 | 선택 |

## 통합

| 결정 항목 | 질문 | 선택지 예시 | 기본 태그 |
|----------|------|-----------|----------|
| 연동 패턴 | 외부 시스템과 어떻게 통신? | 동기 호출, 비동기 메시지, 이벤트 구독 | 필수 |
| 재시도 전략 | 외부 호출 실패 시 재시도 방법은? | 고정 간격, 지수 백오프, DLQ 전송 | 필수 |
| 회로 차단기 | 연쇄 장애 방지 임계값은? | 실패율 50%/10초, 반열림 시도 간격 | 준필수 |
| 데이터 매핑 | 외부 스키마와 내부 모델 변환은? | Anti-corruption layer, 공유 커널, 직접 매핑 | 준필수 |
| 타임아웃 | 외부 호출 최대 대기 시간은? | 연결 3초, 읽기 10초, 전체 30초 | 선택 |

## 사용 방법

1. Phase 3에서 각 모듈의 Decision Card 작성 시 해당 범주의 항목을 참조
2. 기본 태그는 시작점 — 프로젝트 맥락에 따라 승격/강등 가능
3. 이 레퍼런스에 없는 결정 항목도 프로젝트 고유 요구에 맞게 추가 가능
