# Software Component Catalog

> Phase 2 컴포넌트 맵 작성 시 참조하는 소프트웨어 구성요소 카탈로그.
> 프로젝트에 해당하는 항목을 선별하여 사용. 전부 필요하지 않음.

## 범주별 컴포넌트

### 1. API / 인터페이스

| 컴포넌트 | 설명 | Core 경향 |
|----------|------|-----------|
| REST API | HTTP 기반 자원 중심 인터페이스 | Core |
| GraphQL | 쿼리 기반 유연한 데이터 조회 | Core |
| WebSocket | 양방향 실시간 통신 | Core/Support |
| CLI | 명령줄 인터페이스 | Support |
| SDK/클라이언트 라이브러리 | 외부 개발자용 래퍼 | Support |

### 2. 데이터

| 컴포넌트 | 설명 | Core 경향 |
|----------|------|-----------|
| DB 스키마 / 데이터 모델 | 영속 데이터 구조 정의 | Core |
| 캐시 계층 | 읽기 성능 최적화 (Redis, Memcached 등) | Core/Support |
| 메시지 큐 | 비동기 작업 처리 (Kafka, RabbitMQ 등) | Core/Support |
| 파일/객체 스토리지 | 비정형 데이터 저장 (S3, GCS 등) | Support |
| 검색 엔진 | 전문 검색 (Elasticsearch 등) | Support |

### 3. 비즈니스 로직

| 컴포넌트 | 설명 | Core 경향 |
|----------|------|-----------|
| 도메인 서비스 | 핵심 비즈니스 규칙 구현 | Core |
| 워크플로 엔진 | 다단계 프로세스 오케스트레이션 | Core/Support |
| 규칙 엔진 | 동적 비즈니스 규칙 평가 | Support |
| 배치 처리 | 대량 데이터 일괄 처리 | Support |
| 스케줄러 | 주기적 작업 실행 | Support |

### 4. 인증 / 보안

| 컴포넌트 | 설명 | Core 경향 |
|----------|------|-----------|
| 인증 (AuthN) | 사용자 신원 확인 | Core |
| 인가 (AuthZ) | 권한 기반 접근 제어 | Core |
| 감사 로그 | 보안 이벤트 기록 | Support |
| 암호화 / 키 관리 | 데이터 보호, 비밀 관리 | Support |
| 보안 게이트웨이 | WAF, DDoS 방어, IP 제한 | Support |

### 5. 인프라 / 운영

| 컴포넌트 | 설명 | Core 경향 |
|----------|------|-----------|
| 배포 파이프라인 (CI/CD) | 빌드-테스트-배포 자동화 | Support |
| 모니터링 / 메트릭 | 시스템 상태 관측 | Support |
| 로깅 | 구조화된 로그 수집/조회 | Support |
| 알림 | 이상 탐지 시 통보 | Support |
| 서비스 디스커버리 | 동적 서비스 위치 해석 | Support |

### 6. 통합

| 컴포넌트 | 설명 | Core 경향 |
|----------|------|-----------|
| 외부 API 연동 | 서드파티 서비스 호출 | Support |
| 이벤트 버스 | 서비스 간 이벤트 기반 통신 | Core/Support |
| 데이터 동기화 | 시스템 간 데이터 일관성 유지 | Support |
| API 게이트웨이 | 라우팅, 인증, 속도 제한 통합 | Support |
| 웹훅 | 외부 이벤트 수신 | Support |

## 사용 방법

1. Phase 2에서 이 카탈로그를 참조하여 프로젝트에 필요한 컴포넌트를 선별
2. "Core 경향"은 참고용 — 실제 분류는 프로젝트의 Primary Outcome 기준 VCT로 판단
3. 카탈로그에 없는 컴포넌트도 자유롭게 추가 가능
