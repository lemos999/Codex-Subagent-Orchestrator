# 프로젝트 문서 출력 형식

> Phase 5에서 생성하는 문서의 표준 형식.

---

## 1. CLAUDE.md (프로젝트 지침)

```markdown
# [프로젝트명] — 프로젝트 지침

## 역할
당신은 [게임 장르]의 게임 디렉터입니다.
기획 전문이며, 재미가 모든 판단의 최우선입니다.

## 기획 방향
- [공통 원칙 참조: game-design-director SKILL.md]
- [프로젝트 특화 원칙]

## 검증 프로토콜
검증 4단계(오류→모순→맥락→구현)를 따르되, 3단계(맥락) 파라미터:
- 세계관: ___
- 톤앤매너: ___
- 고증: ___

## 파트 분류
| 파트 | 담당 시스템 |
|------|-----------|

## DNA 요약
- 핵심 감정: ___
- 코어 루프: 마이크로/미들/매크로
- USP: ___

## 운용 규칙
- `_confirmed/`에 쓸 수 있는 건 [역할]뿐
- 태그: ✅ 확정 / ⚠️ 미결 / 🔵 방향확정(수치 미정)
- ⚠️/🔵은 `_confirmed/` 진입 불가
```

## 2. _confirmed/ 파일 형식

```markdown
# [시스템명]

> 최종 갱신: [YYYY-MM-DD]
> 주관 파트: [파트명]
> 참조 파트: [파트명1, 파트명2]

## ✅ [항목명]
[수치 또는 규칙 또는 조건]

## ✅ [항목명] — Step-by-Step 구동 로직

| 단계 | 입력 | 시간 | 상태 제어 |
|------|------|------|----------|
| Step 1 | [입력] | [시간] | [상태 변화] |
| Step 2 | [입력] | [시간] | [상태 변화] |

## ✅ [스탯/수치 테이블]

| 항목 | 수치 | 비고 |
|------|------|------|

## ✅ 시스템 인터페이스

| 연결 시스템 | 입력 (값, 범위) | 출력 (값, 범위) | 실패 시 기본값 |
|------------|---------------|---------------|-------------|

## ✅ 모순 검증 통과 항목

| 모순 | 해결 |
|------|------|

## 🔵 [미확정 항목] (기한: YYYY-MM-DD)
방향: ___
```

## 7. parts/ 파일 형식

```markdown
# [파트명] 기획서

> 최종 갱신: [YYYY-MM-DD]

## 담당 시스템
- [시스템명1] (확정: → _confirmed/[시스템명1].md)
- [시스템명2] (확정: → _confirmed/[시스템명2].md)

## ⚠️ 미결 사항
| 항목 | 상태 | 의존 | 기한 |
|------|------|------|------|

## 설계 노트
[_confirmed/에 들어가지 않는 설계 의도, 대안 비교, 참고 자료 등]
```

---

## 태그 범례

| 태그 | 의미 | `_confirmed/` 진입 |
|------|------|-------------------|
| ✅ | 확정 | 가능 |
| ⚠️ | 미결 | **불가** |
| 🔵 | 방향 확정, 수치 미정 | **불가** (기한 명시 필수) |
| ★ | 코딩 전 필수 확정 | Phase 3 인터뷰 대상 |
| ▲ | 코딩 후 1~2주 내 | Phase 3 인터뷰 대상 |
| ○ | 개발 중 확정 가능 | 선택적 |
