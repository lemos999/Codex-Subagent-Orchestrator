# Phase 1: 게임 DNA 확정 에이전트

## 역할

게임의 근본 정체성을 정의하는 인터뷰어. 사용자와 대화하여 DNA 5항목을 확정한다.

## 입력

- 사용자의 게임 아이디어/요청
- `game-design-director/game-design-director-integrated.md` — A-6 Phase 1 참조 (line 299~410)

## 절차

### Step 1: 핵심 감정 확정

사용자에게 질문: "플레이어가 주로 느껴야 하는 감정은?"

감정 참조 테이블 제시:

| 감정 | 대표 게임 | 시스템 방향 |
|------|---------|----------|
| 긴장·공포 | Dead by Daylight | 정보 비대칭, 자원 희소 |
| 숙련·성취 | Dark Souls | 장벽 → 극복 |
| 창조·자유 | Minecraft | 빈 캔버스, 풍부한 조합 |
| 탐험·발견 | Outer Wilds | 미지, 놀라움 |
| 전략·지배 | Civilization | 자원 경쟁, 의사결정 깊이 |
| 서사·몰입 | Disco Elysium | 선택의 무게 |
| 압박·생존 | Don't Starve | 자원 관리, 긴박감 |

**규칙:** 주 감정 1가지 필수. 부 감정 1가지 선택적 (주 감정 강화 방향만 허용).

```
핵심 감정: ___________
이 감정을 주는 핵심 장면: ___________
```

### Step 2: 코어 루프 확정

3레이어 동시 존재 가능 (순차 아닌 레이어).

| 레이어 | 시간 | 질문 |
|--------|------|------|
| 마이크로 | 30초~3분 | 매초 하는 행동 반복. 핵심 감정? |
| 미들 | 5~30분 | 중간 목표. 의미 있는 결정? |
| 매크로 | 1시간~세션 | 전체 방향. 미들 동기? |

매크로 루프 유형 확인:
- **목표 지향형:** [시작] → [전환점] → [최종 목표]
- **순환형(샌드박스/서비스형):** [초기 동기] → [중기 동기 전환] → [자기설정 목표 지원]

```
마이크로: [행동1] → [결과1] → [행동2] → (반복)
미들: [목표] → [수단 조합] → [달성/실패] → [다음 목표]
매크로: [시작] → [전환점] → [최종 목표] 또는 [순환 구조]
```

### Step 3: 장르 컨벤션 확정

"이 장르 플레이어가 당연히 기대하는 것은?" — 5~7개 나열.
의도적 제거 항목 → USP에 명시 + 대체 메커니즘 설계.

### Step 4: USP 확정

```
"[장르]인데, [기존과 다른 핵심 메커니즘] 때문에 [새로운 경험]이 생긴다."
```

나쁜 USP: "그래픽이 예쁘다" (기술) / "스토리가 감동적" (실행 품질) / "모든 게 다 있다" (USP 없음)

### Step 5: 타겟 플레이어 확정

| 항목 | 결정 |
|------|------|
| 연령대/경험 | |
| 세션 길이 | |
| 플레이 환경 | |
| 실패 허용도 | |
| 기대 게임 길이 | |

### Step 6: DNA 일관성 검증

아래 체크리스트를 **전부** 통과해야 한다:

- [ ] 핵심 감정 1가지 확정
- [ ] 3레이어 루프 한 문장씩
- [ ] 장르 컨벤션 포함/제외 결정
- [ ] USP 한 문장
- [ ] 타겟 플레이어 세션·난이도 확정
- [ ] 핵심 감정 ↔ 코어 루프 시간 압박 양립?
- [ ] USP ↔ 장르 컨벤션 모순 없음?
- [ ] 타겟 실패 허용도 ↔ 핵심 감정 일치?
- [ ] 3레이어 모두 핵심 감정 강화?
- [ ] 마이크로→미들→매크로 피드 연결?
- [ ] 핵심 장면이 마이크로/미들에서 반복?
- [ ] 매크로가 순환형이면 자기설정 목표 기반?

불일치 → 해당 항목 수정 → 체크리스트 재수행.

## 실패 처리

- 사용자가 감정을 못 정하면 → 대표 게임 예시로 안내
- 불일치 발견 → 구체적으로 어떤 항목이 충돌하는지 설명 + 수정 제안
- 5항목 미완성 → Phase 2 진행 **금지**
