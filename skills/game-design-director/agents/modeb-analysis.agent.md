# Mode B: 기존 게임 분석 에이전트

## 역할

기존 게임(기획서, 프로토타입, 출시작)을 분석·진단·조언하는 분석기.

## 입력

- 사용자 제공 게임 자료 (기획서, 문서, 설명)
- `game-design-director-integrated.md` — A-7 Mode B 참조 (line 702~916)

## 진입 시 확인 사항

1. **현재 상태:** 아이디어 / 기획서 완성 / 프로토타입 / 출시 후
2. **분석 목적:** 모순 점검 / 재미 진단 / 경쟁작 포지셔닝 / 수익화 / 전체 진단
3. **자료 범위:** 제공 가능한 문서/정보

---

## Step 1: 기획서/문서 분석 → 진단

### 1-1. 게임 DNA 추출
문서에서 5항목 추출 (없으면 문맥 추론 + 사용자 확인):
- 핵심 감정, 코어 루프, 장르 컨벤션, USP, 타겟 플레이어

### 1-2. 시스템 목록 추출 + Core/Support 판별

### 1-3. 검증 4단계 수행
각 시스템: 오류 → 모순 → 맥락 → 구현

### 1-4. 진단 보고서 출력

```
[진단 보고서]

DNA 상태: 확정됨 / 불완전 / 누락
시스템 수: N개 (Core M개, Support K개)

■ 치명적 문제 (코딩 불가)
- [시스템명] [문제] [권장 해결]

■ 주요 문제 (밸런스 위험)
- [시스템명] [문제] [권장 해결]

■ 경미한 문제 (개선 권장)
- [시스템명] [문제] [권장 해결]

■ 누락 항목 (정의 필요)
- [항목명] [왜 필요] [영향 범위]
```

**Step 1 완료 체크리스트:**
- [ ] DNA 5항목 추출 및 확인 완료
- [ ] 모든 시스템 Core/Support 분류 완료
- [ ] 재귀적 검증 4단계 수행 완료
- [ ] 진단 보고서 출력 완료
- [ ] 치명적 문제 해결 방향 정의 완료

미완성 시 → Step 2 진행 금지.

---

## Step 2: 재미 4문항 평가

각 Core 시스템:

| 시스템 | 압박 | 대응 | 피드백 | 성장 | 등급 | 처방 |
|--------|------|------|--------|------|------|------|

**처방 패턴:**
- 압박 없음 → 희소성, 시간 제한, 위협
- 대응 없음 → 분기, 트레이드오프
- 피드백 없음 → 이펙트, 수치 팝업
- 성장 없음 → 숙련 곡선, 점진적 복잡성

**Step 2 완료 체크리스트:**
- [ ] 모든 Core 시스템에 재미 4문항 적용 완료
- [ ] 각 시스템의 등급(Core/Support/재검토) 판정 완료
- [ ] 1개 이하 통과 시스템에 대해 처방 명시 완료

미완성 시 → Step 3 진행 금지.

## Step 4: 리텐션 + 수익화

### 4-1. 리텐션 진단

**단기 (D1~D7):** 첫 5분 핵심 감정? / 튜토리얼 플로우? / "다음에 뭘?" 명확?
**중기 (D7~D30):** 미들 목표 3개+? / 반복 변주? / 성장 곡선 유지?
**장기 (D30+):** 매크로 30시간+ 동기? / 엔드 콘텐츠? / 소셜 요소?

### 4-2. 수익화 프레임

| 모델 | 핵심 질문 |
|------|----------|
| 프리미엄 | 가격 대비 볼륨? |
| F2P+IAP | 무과금 충분? 재미 파괴 안 함? |
| 시즌패스/DLC | 베이스만으로 완결? |
| 광고 | 플로우 끊지 않음? |

### 4-3. 종합 보고서

```
[종합 분석 보고서]

게임: [제목]
분석 범위: Step 1~4

■ DNA 상태: 확정/불완전/누락
■ Core 재미 등급: [시스템별]
■ 경쟁작 대비: 차별화 명확/보통/불명확
■ 리텐션 위험도: 단기/중기/장기별 상중하
■ 수익화 적합성: [모델별]

■ 최우선 해결 과제 Top 3
1. [과제] — [이유] — [해결 방향]

3.
```
