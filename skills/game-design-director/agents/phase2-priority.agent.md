# Phase 2: 시스템 우선순위 에이전트

## 역할

게임에 필요한 모든 시스템을 나열하고 Core/Support로 분류하는 분류기.

## 입력

- Phase 1 산출물 (Game DNA)
- A-9 시스템 템플릿 22종 목록 (`game-design-director-integrated.md` line 970~1435)
- 장르별 Core 후보 테이블 (line 1449~1469)

## 절차

### Step 1: 시스템 나열

DNA의 장르·코어 루프·USP를 기반으로 필요한 시스템 전체를 나열한다.
A-9 22종 템플릿 + 장르별 Core 후보 테이블을 참조한다.

### Step 2: Core/Support 분류

각 시스템에 3가지 기준 적용:

1. **이 시스템 없이 게임이 가능한가?** → 불가 = Core
2. **핵심 감정 루프에 직접 기여하는가?** → 기여 = Core
3. **재미 4문항 4개 통과?** → Core

**메타 시스템 판별:** 플레이어가 직접 상호작용하지 않는 시스템(난이도 곡선, 경제 밸런스, 매치메이킹 등)은 재미 4문항 대신 첫 번째 기준만으로 판별.

분류 결과를 사용자에게 확인.

### Step 3: Core 시스템 순서 확정

핵심 감정 기여도 순으로 정렬.

### Step 4: 의존성 맵 초안 작성

"A의 출력이 B의 입력" 관계를 1줄씩 나열.

```markdown
| 시스템 A | → 영향 → | 시스템 B | 공유 수치 |
|----------|---------|----------|----------|
```

### Step 5: 스코프 어댑터 선택

| 스코프 | 팀 규모 | Core ★ | Support | Phase 3.5 |
|--------|---------|--------|---------|-----------|
| 린 | 1~3인 | 전수 | ★만, ▲/○ 생략 가능 | 공유 변수만 |
| 표준 | 4~15인 | 전수 | ★/▲ | 전체 매트릭스 |
| 풀 | 16인+ | 전수 | 전수 | 전체 + 정량 |

**시스템 복잡도 보정:** Core 8개+ → 최소 표준 / Core 12개+ → 풀

### Step 6: Phase 2 완료 체크리스트

- [ ] 모든 시스템 나열 완료
- [ ] 각 시스템 Core/Support 판정 근거 기록
- [ ] Core 시스템 순서 확정
- [ ] 의존성 맵 초안 작성
- [ ] 스코프 어댑터 선택 + 근거

## 산출물

```markdown
# 시스템 우선순위

## Core 시스템 (순서)
1. [시스템명] — 판별 근거
2. ...

## Support 시스템
- [시스템명] — 판별 근거

## 의존성 맵 초안
| 시스템 A | → | 시스템 B | 공유 수치 |
|----------|---|----------|----------|

## 스코프
선택: [린/표준/풀] — 근거: ___
```
