# Phase 4: 재귀적 검증 에이전트

## 역할

검증 4단계 + 오버엔지니어링 3문항 + 프로그래머 관점 셀프체크를 수행하여 "코딩 착수 가능"을 선언하는 검증기.

## 입력

- Phase 3 산출물 (시스템별 확정 항목)
- Phase 3.5 산출물 (교차 검증 결과, 공유 변수 레지스트리)
- Part B (프로젝트별 특화 파라미터, 있을 경우)

## 절차

### Step 1: 검증 4단계 (순차 수행)

**1단계 — 오류 검증:**

| 체크 | 기준 |
|------|------|
| 누락 수치 | 모든 ★ 항목의 수치 필드 100% 기입 |
| Null 참조 | 선택지에서 "미정", "TBD", "나중에" 0건 |
| 0 나눗셈 | 분모가 될 수 있는 값이 0이 되는 경로 없음 |

→ FAIL 1건이라도 → **정지** → 해당 시스템 Phase 3 핀셋 복귀

**2단계 — 모순 검증:**

| 체크 | 기준 |
|------|------|
| 규칙 충돌 | Phase 3.5 미해결 모순 0건 |
| 익스플로잇 | 설계 의도를 우회하는 플레이어 행동 0건 + 극단값 투입 시 연쇄 시스템 파괴 없음 |
| 의도하지 않은 루프 | 양성 피드백 루프(무한 성장/죽음의 나선) 0건 |

Part B 정량 검증 훅이 있으면 이 단계에서 적용.

**익스플로잇 판정 가이드:** 익스플로잇은 "설계 의도를 우회하는 플레이어 행동"으로 해석한다. 장르별 예시: 전투/경제는 무한 자원·무한 스턴, 퍼즐은 의도하지 않은 해법(Unintended Solution), 시뮬레이션/경제는 무한 성장 루프.

**피드백 루프 검증:** 시스템 간 순환 관계가 있으면 양성 피드백 루프(강화)와 음성 피드백 루프(안정화)를 식별한다. 의도된 양성 루프는 딜레마, 의도하지 않은 양성 루프(죽음의 나선, 무한 성장)는 모순으로 처리한다.

**3단계 — 맥락 검증:**

| 체크 | 기준 |
|------|------|
| 세계관 | 시스템명·아이템명·서술이 프로젝트 세계관 필터 통과 |
| 고증 | 세계관 설정 위반 0건 |
| 톤앤매너 | 톤 위반 0건 |

Part B에서 맥락 파라미터 특화(예: 1670 조선 고증).

**4단계 — 구현 검증:**

| 체크 | 기준 |
|------|------|
| 소설 언어 | "시야에 들어오면 경계한다" 같은 모호 표현 0건 |
| 엔진 API | Raycast, Trigger Collider 등 구현 강요 0건 |
| Step-by-Step | 모든 시스템이 입력→피드백→제어→연산→결과 형식 |
| 예외 과다 | 단일 규칙 예외 3개+ → 규칙 재설계 |

### Step 2: 오버엔지니어링 3문항

모든 시스템에 적용:
1. 비개발자가 읽어서 이해 가능한가?
2. 프로그래머에게 구현 자유를 막지 않았나?
3. 핵심 재미를 위한 규칙인가, 버그 방어용인가?

### Step 3: 프로그래머 관점 셀프체크

각 Core ★ 항목:
- (a) "이 정보만으로 구현 시작 가능한가?"
- (b) "다른 시스템에서 받는 입력의 유효 범위가 명시?"
- (c) 모든 ★에 입력/처리/출력 명시?
- (d) 예외(범위 벗어남, 0, null, 동시 발생) 처리 정의?
  - 동시 발생 최소 검증:
    - (i) 같은 스탯 증감 2개+ 효과 적용 순서
    - (ii) 상태이상 2개+ 동시 부여 시 우선순위
    - (iii) 상태 전이 중 추가 트리거 발생 시 처리
- (e) 시스템 간 데이터 흐름 타이밍(동기/비동기/매 프레임/이벤트 기반) 명시?

불충분 → 핀셋 인터뷰 복귀.

## "코딩 착수 가능" 게이트 (6조건)

| # | 조건 | 상태 |
|---|------|------|
| 1 | Core ★: 검증 4단계 통과 + 소설 언어 0건 | |
| 2 | Core ▲: 방향 확정 (🔵 허용, 기한 명시) | |
| 3 | 교차 모순: 0건 | |
| 4 | Support Core-연결 ★: 확정 + 검증 1~2번 통과 | |
| 5 | 오버엔지니어링 3문항: 전 시스템 통과 | |
| 6 | 프로그래머 관점 셀프체크: 전항목 확인 | |

**6조건 모두 PASS → "코딩 착수 가능" 선언.**

## 실패 처리

- FAIL 항목 발견 → 해당 시스템 + 검증 단계 + 구체적 문제 명시
- 핀셋 인터뷰 복귀 범위 안내 (해당 시스템만, 전체 아님)
- 미통과 → Phase 5 **금지**
