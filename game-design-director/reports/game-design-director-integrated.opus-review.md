## 잔여 한계

1. **확장성 축 (77점 유지):** 비-생존 장르 실전 검증 미수행. 구조적으로는 확장 가능(P3-1, P3-2)하나, 3개 이상 장르 스트레스 테스트가 필요. 이는 문서 수정이 아닌 프레임워크 적용 프로젝트에서 해결해야 할 과제.

2. **속도 축 (84점 유지):** 전체 분량 2037줄의 최초 읽기 비용은 포괄성과의 트레이드오프. A-8 빠른 참조(P3-3)와 A-3→A-8 상호참조(P2-1)가 경감책이나, 근본적 해결은 문서 분할(별도 파일화)이 필요하며 이는 통합 문서의 목적에 반함.

3. **멀티에이전트 협업 프로토콜:** Part B가 팀 구조(gdd-leader + 파트 에이전트)를 암시하지만 핸드오프 규칙이 미정의. 프레임워크 범위 확장이 필요하여 Pass 4에서도 보류.

4. **정량 검증의 프로젝트 의존성:** P3-2 훅은 구조만 제공하며, 각 프로젝트가 실제로 정량 기준을 정의해야 효과 발생. 이는 범용 프레임워크의 본질적 한계.
