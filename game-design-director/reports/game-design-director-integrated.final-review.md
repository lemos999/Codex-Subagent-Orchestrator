## 적용된 변경 사항 (Pass 2)

### P2-1: A-3/A-8 상호참조로 중복 읽기 비용 해소

- **변경 내용:**
  - A-3 끝에 `(이론 배경·적용 예시·약점 패턴: → A-8 재미 이론 참조서)` 추가
  - A-8의 재미 4문항 코드블록(정의 반복)을 `A-3의 재미 판단 4문항을 아래와 같이 적용한다. (정의: → A-3 참조)` + 적용 형식 인라인 안내로 대체
- **5축 기여:** 속도(중복 읽기 비용 제거), 명확성(A-3=정의, A-8=이론+적용의 역할 분담 명시)
- **Part C 근거:** Part C의 시스템 기획서 작성 시 A-3과 A-8을 모두 참조해야 했으나, 동일한 4문항이 두 곳에 기술되어 어느 것이 원본인지 불명확했음. 이제 A-3이 유일한 정의 소스.
- **잔여 과제 해소:** F-A6 (Pass 1에서 보류됨) → 경량 상호참조 방식으로 해소.

### P2-2: B-6 태그 전환 규칙 정의

- **변경 내용:** B-6에 태그 전환 테이블(⚠️→✅, 🔵→✅, ✅→⚠️) 및 차단 규칙(⚠️/🔵 → _confirmed/ 진입 불가) 추가
- **5축 기여:** 빈틈(태그 전이 조건 미정의 해소), 검증력(Phase 5 태그 감사에서 미확정 항목 시스템적 차단), 명확성(태그 운용 모호함 해소)
- **Part C 근거:** Part C에서 ⚠️/🔵 태그가 실전 사용되지 않은 것은 전환 규칙이 없었기 때문. C-2~C-6의 모든 ★ 항목이 ✅로 확정되었으나, 이 전환이 어떤 조건에서 이루어지는지 암묵적이었음.
- **잔여 과제 해소:** F-B4 (Pass 1에서 보류됨) → 전환 테이블+차단 규칙으로 해소.

### P2-3: Phase 3.5 교차 검증 우선순위 규칙

- **변경 내용:** Phase 3.5에 `교차 검증 우선순위: 공유 변수 레지스트리에서 참조 시스템 수가 많은 변수부터 검증한다. 3개 이상 시스템이 공유하는 변수는 최우선 검증 대상이다.` 추가
- **5축 기여:** 속도(대규모 시스템에서 검증 우선순위 제공), 검증력(고위험 변수 우선 검증)
- **Part C 근거:** C-7에서 HP, 체온, 시체 존속 시간 등 3개 이상 시스템에 걸친 공유 변수가 가장 많은 교차 모순을 유발했음. 우선순위 없이 전수 검증은 비효율적.

### P2-4: Phase 6 Tier 1 빠른 경로

- **변경 내용:** Phase 6 변경 규모 분류 이후에 `Tier 1 빠른 경로: 공유 변수 레지스트리에 해당 수치가 없으면 의존성 맵 참조 없이 해당 시스템 밸런스 확인만으로 완료한다.` 추가
- **5축 기여:** 속도(비공유 변수 변경 시 불필요한 의존성 맵 참조 생략), 빈틈(Tier 1 정의에 이미 "공유 변수 아님"이 있었으나 절차 1단계에서 의존성 맵 참조를 요구하는 모순 해소)
- **Part C 근거:** C-2 전투 시스템의 단일 무기 데미지 수치 조정 같은 Tier 1 변경이 의존성 맵 전체를 참조해야 하는 것은 과도함. 공유 변수 레지스트리가 이미 Phase 3.5에서 생성되므로 빠른 판별 가능.

## 거부된 아이디어

### Pass 1 잔여 과제에서 거부된 항목

1. **F-A5 (22종 템플릿에 소셜/UGC/접근성 추가)** — 거부.
   - 사유: 실전 프로젝트에서 해당 장르가 Core로 식별되기 전까지 템플릿 추가는 예측적 확장. Pass 2의 "투기적 추가 금지" 원칙에 위배. 현재 22종으로 대부분 장르를 커버하며, 필요 시 A-9의 장르 테이블 확장 주석(F-U2)이 안내 역할을 함.

2. **F-U3 (핵심 감정 테이블 7가지 확장 주석)** — 거부.
   - 사유: "하나만 선택" 원칙이 이미 명확하여 혼동 위험 낮음. "자유 정의 가능" 주석 추가는 1줄이지만, 근거가 되는 실전 피드백이 없는 상태에서의 추가는 투기적.

3. **F-M4 (밸런스 검증 도구/방법론 부재)** — 거부.
   - 사유: 밸런스 검증 방법론은 장르/시스템마다 크게 다름. 범용 프레임워크에 특정 방법론을 고정하면 확장성이 저하됨. Phase 4의 "오류: 누락 수치? 0 나눗셈?"이 최소 검증을 제공하며, Phase 3.5의 교차 검증이 시스템 간 밸런스 검증을 보완함.

4. **Phase 3.5의 실전 검증** — 거부 (문서 변경 불필요).
   - 사유: Phase 3.5는 실전 적용 전 단계에서 구조적 검증만 가능. 실전 검증은 프레임워크 적용 프로젝트에서 수행해야 하며, 문서에 "실전 검증 필요" 주석을 다는 것은 정보가 아닌 면책조항. Pass 2에서 추가한 우선순위 규칙이 병목 위험을 경감함.

5. **Phase 6 변경 관리의 실전 검증** — 거부 (문서 변경 불필요).
   - 사유: 위와 동일. Pass 2에서 추가한 Tier 1 빠른 경로가 과도한 절차의 주된 우려(소규모 변경에 5단계 전체 적용)를 해소함.

## 잔여 한계

1. **확장성 축 미개선 (72점 유지):** 소셜/UGC 템플릿, 감정 테이블 확장 등이 보류되었으나, 이는 실전 피드백 없이 투기적 추가를 피하는 의도적 결정. 88점에서 90점 이상으로 진행하려면 실전 프로젝트 적용 후 피드백 기반 확장이 필요.

2. **밸런스 검증 방법론 부재:** Phase 4가 "오류 검증"에 한정되어 있고 구체적 밸런스 계산 방법(DPS 시뮬레이션, 극단값 시나리오 등)은 미정의. 이는 범용 프레임워크의 본질적 한계이며, Part B(프로젝트 특화)에서 보완하는 것이 적절.

3. **Phase 3.5/6의 실전 미검증:** 구조적으로는 건전하나, 실제 프로젝트에서의 병목 여부는 확인 불가. 우선순위 규칙(P2-3)과 빠른 경로(P2-4)가 경감책이나 완전한 해소는 아님.

4. **A-8 재미 이론 참조서의 분량:** A-8은 여전히 긴 참조서이나, 참조서 특성상 전체 통독이 아닌 필요 시 참조 용도. 상호참조(P2-1)로 진입 비용은 감소했으나 분량 자체는 유지.

## Pass 1 대비 개선 근거

### 구체적 개선 증거

1. **중복 제거 (P2-1):** A-8에서 A-3의 4문항을 코드블록으로 반복하던 6줄이 2줄 참조로 대체됨. 동일 정의의 이중 유지보수 위험 해소. A-3이 유일한 정의 소스(Single Source of Truth)로 확립됨.

2. **태그 운용의 시스템화 (P2-2):** Pass 1에서 태그(✅/⚠️/🔵)는 "라벨"에 불과했으나, 전환 조건 테이블과 차단 규칙 추가로 "상태 머신"이 됨. Phase 5 태그 감사가 ⚠️/🔵의 _confirmed/ 진입을 시스템적으로 차단 가능.

3. **검증 효율화 (P2-3, P2-4):** Pass 1에서 Phase 3.5는 "모든 공유 수치를 검증"하라는 지시만 있었으나, 우선순위 규칙으로 3개 이상 시스템 공유 변수를 최우선 검증 대상으로 지정. Phase 6은 모든 변경에 동일 비용을 부과했으나, Tier 1 빠른 경로로 비공유 변수 변경을 1단계로 축소.
